In this lecture, you are going to learn about the model we're going to use to represent the machine on which we will be executing our algorithms. In the rest of the course, we are going to use this model to analyze the running time and the memory requirements of our algorithms. The model is called the Random Access Machine model. In short, the RAM model. As with any model, the RAM model is a simplified representation of reality. Our reality is a computer. A computer is a complex creature so you really don't want to have to deal with all details inside it. Instead, you can develop a model of it. The model must be complete enough to capture these aspects of the computer you need to analyze, but simple enough to allow you to do the analysis. This model is going to be good enough for us in this course. So here we have a central processing unit which can read and write data from the external environment. For example, the CPU can read data entered by the user using a keyboard, or it can display data on a screen. The CPU also has access to a storage unit, where it can write and read stored data. Using this very simple model, we are going to estimate the running time of an algorithm according to the following four assumptions. Firstly, let's assume that we have a machine with a single CPU, this means that instructions are executed sequentially. Secondly, let's assume that each simple operation takes one unit of time. Simple operations include numerical operations such as subtraction, sum, and multiplication. Notice here how this really is a simplification of reality, because you know that a multiplication is made up of several sums. In reality, is not possible that a multiplication will take the same time as a sum. However, in this model, we are going to assume that every numerical operation takes exactly one unit of time. Other types of simple operations that take one time unit are control instructions such as conditional branching, if else; calling a function, not executing the function itself, just calling it; writing or reading data from memory and returning from a function. Certainly, let's assume that loops and functions are not simple operations. They take longer to execute because they are made up of several simple operations. So when you face a loop or a function in a code in your algorithm, you will have to go inside them and see how many simple operations they are made up of. Finally, let's assume that memory is unlimited. That is, we have as much memory as is needed in this model. Additionally, there's no memory hierarchy. Usually in a real computer, data that is in the cache memory is accessed much faster than data that is in the RAM memory. In turn, data in the RAM memory is accessed much faster than data stored in the hard disk. In this model, we are going to assume there is only one type of a storage unit and that every access, either to write or read data, is going to take one unit of time. Now, regarding memory consumption, the RAM model makes just one assumption. Every simple variable, no matter its type, uses one memory position. Therefore, an integer value uses only one memory position, but an array of N elements uses N memory positions. This is the RAM model.

# Checklist of required skills and knowledge

. Duration: 10 minutes

To successfully complete the Coursework 1 Assignment, you would need to develop skills, concepts, and knowledge in several areas related to algorithms, recursion, sorting and hashing. General skills required for Coursework 1 which you practiced across Topics 1-5 : 1. Pseudocode writing and interpretation: Writing clear, correct pseudocode for different algorithms. 2. Complexity analysis: Confidence in determining and comparing time and space complexities. 3. Problem-solving: Ability to choose appropriate algorithms and techniques based on specific problem requirements and constraints. Learning outcomes for Coursework 1: Calculate the time complexity of different algorithms Recommend the best algorithm to solve a specific problem Create an algorithm based on the concept of hash Here is a more detailed list of skills, concepts and knowledge which you need to have in order to complete Coursework 1 successfully: Analysis of Algorithms Required Knowledge: Definitions and interpretations of Big-O, Omega, and Theta notations. Familiarity with concepts such as constant time, linear time, logarithmic time, quadratic time, and exponential time complexities. Key Skills and Concepts: 1. Algorithm analysis techniques Understanding the difference between theoretical (analytical) and empirical (experimental) analysis. Analyse an algorithm by both looking at its design and testing its performance. 2. Pseudocode interpretation Being able to read and interpret pseudocode to analyse the time and space complexity. 3. Time and space complexity calculation Calculating how much time and memory an algorithm will consume based on its pseudocode structure. 4. Growth functions Determining the growth function of an algorithm's runtime or memory use, which describes how resource needs increase with input size. 5. Big-O, Omega, and Theta Notation Using Big-O notation to describe upper bounds of algorithm performance. Using Omega notation to describe the lower bound. Using Theta notation for average case or tight bounds. Recursive Algorithms Required Knowledge: Concepts of base and recursive cases in recursion. Recurrence relations and methods (such as the Master Theorem) for analysing recursive time complexity. Basic and advanced examples of recursion, such as factorial and the Fibonacci sequence. Key Skills and Concepts: 1. Understanding recursion Understanding the fundamental principles of recursion, including base cases and recursive cases. Recognising when recursion is suitable for a problem. 2. Tracing recursive algorithms ing through recursive calls step-by-step to understand how the algorithm executes. 3. Converting iterative to recursive algorithms Writing the recursive equivalent of an iterative algorithm in pseudocode, understanding when and why recursion may be preferable. 4. Time complexity of recursive algorithms Analysing the time complexity of recursive algorithms, particularly using recurrence relations. Comparison Sorting Algorithms Required Knowledge: Fundamentals of each sorting algorithm's process, strengths, and weaknesses. Familiarity with worst-case, average-case, and best-case performance for each sorting method. Understanding of when to use stable vs. unstable sorting algorithms and their impact on sorted data. Key Skills and Concepts: 1. Sorting approaches Understanding the differences between comparison-based sorting algorithms: bubble sort, insertion sort, selection sort, merge sort, and quicksort. Identifying scenarios where each algorithm is optimal or suboptimal. 2. Implementing sorting algorithms Writing implementations for the five sorting algorithms in pseudocode. 3. Time complexity analysis Calculating the best, worst, and average time complexities of each sorting algorithm. Non-Comparison Sorting Algorithms Required Knowledge: Counting sort, radix sort, and bucket sort and their optimal use cases. Constraints and assumptions for non-comparison sorts (e.g., range limitations in counting sort). Key Skills and Concepts: 1. Understanding non-comparison sorting Differentiating non-comparison sorts like counting sort, radix sort, and bucket sort from comparison-based sorts. Recognising the specific types of data and conditions where non-comparison sorts excel. 2. Implementing non-comparison sorting algorithms Writing implementations for counting sort, radix sort, and bucket sort. 3. Analysing time complexity Determining the time complexity for each non-comparison sort, considering the constraints and characteristics of each algorithm. Hashing Required Knowledge: Basics of hash functions, including what makes a function good for hashing. Linear probing and other collision resolution techniques, their advantages, and trade-offs. Using hash tables and potential challenges like clustering and load factor management. Key Skills and Concepts: 1. Understanding hashing The concept of hashing, hash functions, and their applications in data storage and retrieval. 2. Collision resolution methods Different methods for resolving hash table collisions, such as linear probing, chaining, and quadratic probing. Implement these methods in a basic hash table structure with collision resolution using linear probing.

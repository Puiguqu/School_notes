During the first half we studied algorithms, we learned how to analyze algorithms and understand recursive algorithms and their complexity. We have studied different comparison and non-comparison sorting algorithms, and learned about hashing for fast search. During this second half, we will study data structures. Data structures are data containers, where data is organized in a specific way. The simplest ones have a linear organization of data, and they will be the first ones we will study. They are linked lists, stacks, and queues. You already know two data structures with a linear organization, the one-dimensional arrays and hash tables. All linear data structures store one number after another following a line, but they differ in the way they yet access to those elements. We will then move onto non-linear data structures. The first type of non-linear data structures we will study have a hierarchical organization, they are trees and heaps. Finally, we will study a second type of non-linear data structures, graphs. Before starting with the linear data structures, let's define what a data structure is. A data structure is a container of data where data is organized in a very specific way. For example, lists are organized in a linear manner, whereas trees and heaps are organized in a hierarchical way. Finally, every data structure has a set of specific operations or functions to access and manipulate the data stored in it. For example, the three main functions associated to a hash table are insert, search, and remove. We will see what specific functions are used for each data structure when we studied them individually. So data structures and algorithms are closely related. You cannot have a data structure without a series of algorithms helping you to operate the data stored in the data structure.

For this reason from now on every element of a list will be referred to as a node. In the pseudocode when we refer to the data part of the note, we will write node arrow data. And when we refer to the address of the next element, we will write node arrow next. Now let's start understanding the pseudocode of the function insert. Let's assume we already have a linked list with four nodes on it. The data content of the nodes is from left to right, 3, 7, 2, and 9. When you are going to insert a new node in a list, the first step is creating the new node with the corresponding data. Let's say we are going to insert a new node with a number 4 in the data part. So we ask for the system to allocate memory to that new node. Let's assume that the system has allocated the memory address 0xF to this new node. When the new node is created, it's not linked to the list in any way. No other node is pointing to the new node, and the new node is not pointing to any node in the list either. We have to do that job ourselves when inserting the new node. Now comes step 2, where the linking task must be done. That is one of the nodes or the head must point to the new node, and the new node must point to some node of the list or to null. Depending on where this new node is going to be inserted, you need to perform different linking tasks. Let's assume we are going to insert the node at the beginning of the list. In that case, the first part of step 2 will be making the head to point to the new node. However if we do that, we lose the information already stored in the head, the address of node with number 3 on it in this case. That is no one has the address of the first element on the list. So we cannot start like that. We'll start by making the new node storing number 4 to point to the first element of the list, the node storing number 3. We can do that because the information of the memory address of the node with number 3 is in the head. So we copy the information stored in the head, the memory address of the node with number 3, to the address part of the new node. We do that with the instruction newNode arrow next equal head. With this, the head and the new node are both pointing to number 3. And now we can make the head to point to the node storing number 4. We do that with the instruction head equal newNote. We rearrange the visualization of the list to look like a line again. And we have finished the insertion of the new node. The linked list now has the nodes 4, 3, 7, 2, and 9. Let's review the pseudocode now. On the screen you can see the pseudocode of the function insert when the insertion is done at the beginning of the list. This instruction creates the new node, asking for memory space for the data part which stores number 4, and the address part which is stores NULL when just created. The next instruction copies the content of head to the address part of the new node. This makes the new node storing number 4 to point to the first node of the list storing number 3. Finally, we update the content of head to point to the new node. Now let's see how this pseudocode works when we start with an empty list, that is a list where the content of the variable head is equal to NULL. The constructor asks for memory space and stores number 3 in the data part of the new node, and NULL in the address part. Let's assume that this node has been created in the memory address 0xA. Now the address part is updated to point to the first element of the list, that is updated with the content of head. However, there are no elements in this list yet. So head stores NULL. As the address part of the new node is already storing NULL, this instruction does not produce any change in the address part of the new node. With this we've finished the simplest version of the insert function for a linked list. This version of insert has a time complexity equal to Theta(1) as each instruction takes constant time to be executed. There are two other versions of insert. There is one that inserts the new element at the end of the list. In that case, after creating the new new node, you must make the last element of the list to point to the new node. In this example the last element in the list was the node storing number 3. This node must now point to the new node storing number 9. The other version inserts the new element in an arbitrary position in the list, in which case the new element must be given a space between two elements of the list. This is done by making the new node to point to the next element in the list, and the new node being pointed at by the previous element in the list. In this example the new node storing number 4 has been inserted between the nodes storing numbers 3 and 9. Therefore the node store in number 3 must point to the new node, and the new node must point to the node storing number 9.

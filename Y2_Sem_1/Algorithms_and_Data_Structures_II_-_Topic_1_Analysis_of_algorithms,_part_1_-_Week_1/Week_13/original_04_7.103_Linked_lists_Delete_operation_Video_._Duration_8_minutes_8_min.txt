By doing so, number 7 is not referenced by any element of the list, and there is no way of getting to it. In some programming languages, any element that does not get referenced is automatically eliminated from memory. In some other languages, you must take care of this task yourself. In either case, the resulting list does not include the node with number 7 anymore. To understand the part of the algorithm that performs the task of eliminating one node of the list, let me introduce two variables we will need to use. The variable prev, which points to the element previous to the element to be deleted. The variable tmp, which points to the current element, that is, the element to be deleted. Tmp is an abbreviation of the word temporal, since it's a variable that is used temporarily to traverse the list. First, we have to copy the address part of the element pointed to by tmp to the address part of the element being pointed to by prev. The node with number 7 is not being pointed to by any other node. To make the element with number 7 disappear from memory, there are two options depending on the programming language. Some languages asked for an explicit instruction to release that memory position, and some other languages do it automatically when an element is no longer pointed out by any other element in the program. In either case, the resulting effect is that the element being deleted disappears from memory. A special case occurs when the node to be deleted is the first node of the list.

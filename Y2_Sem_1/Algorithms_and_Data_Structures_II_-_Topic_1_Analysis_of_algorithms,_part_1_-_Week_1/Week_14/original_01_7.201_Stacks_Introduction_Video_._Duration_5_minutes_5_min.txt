Some screen readers may require using CTRL in conjunction with the alt key Hello. In this lecture, we will study a second linear data structure, the stack. A stack data structure works very much like a physical stack. Objects can only be inserted at the top of the stack. Unlike linked list where you can insert an element at any position of the list. With a stack, you can only insert at the top. The insertion operation in a stack is called a push. Analogously, objects can only be removed from the top of the stack. The removal operation is called pop. If you want to query the content of the stack, the only element you can look at is the one at the top of the stack. The operation that returns the value of the element at the top of the stack is called peek. Finally, the operation is empty, returns true if the stack is empty, and false if it has at least one element. So a stack is a linear data structure with important constraints to the access of the data stored in it. You can only insert, remove, or look at an element if that element is stored at the top of the stack. What are the uses of a stack? Let me show you two typical uses for these data structure. One very common use is the verification of balanced pairs of elements, such as parenthesis which must come in pairs. For example, when your compiler alerts you to an error relating to the lack of a closing parenthesis, it has most probably used a stack data structure to check this. How does the compiler use a stack to detect the lack of parenthesis? Well, every time an opening parenthesis is found, an element is pushed into the stack, and every time a closing parenthesis is found, an element is removed from the stack. So in this line, nothing is done regarding that, but in this line, an element is pushed into the stack, signaling the presence of an opening parenthesis. In the next lines, no parenthesis is found until we got to this line. Here a new opening parenthesis has been found and therefore another element is pushed into the stuck. In the next two lines, no parenthesis are found and then in this line a closing parenthesis is found. In this case, a pop operation is performed. By removing one of the elements representing an opening parenthesis, we match one closing parenthesis with one opening one. Now we get to the end of the file and we execute the operation is empty on the stack. If the parenthesis were balanced, then this operation will return true. However, in this case, it's going to return false, signaling that there is a missing closing parenthesis. In that case, an error alert is launched, so the programmer is made aware that there is a missing parenthesis in their code. Another use of stacks is undoing actions in inverse chronological order, as you do when you press "Control-set" in your text editor to undo your last changes in a document. To offer you the possibility of undoing your last actions, the system keeps a stack where your actions are stored. So you execute Action 1 and that gets pushed into the stack. Then you execute Action 2 and it's pushed into the stack. The same happens for actions 3, 4, 5 and then you realize you actually need to undo the last two actions. Because the system has stored them in a stack, it's easy to remember the last actions in reverse chronological order. You just need to pop them from the stack. So with the first pop, you recover the meaning of Action 5, so you can undo it, and with the second pop, you recover the meaning of Action 4 so you can undo it.

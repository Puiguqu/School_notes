Most modern programming languages have libraries were a stack and its operations are already implemented. Or they directly have the operations of a stack built in the set of basic instructions. If you want to use a standard stack and your favorite programming language already has the operations for it, go for it. However, sometimes you need to modify a standard data structure, and that is when knowing how to implement it comes in handy. A stack is a linear data structure. So it can be implemented using other linear data structures, an array, or a linked list. First notice how the linked list only creates as many nodes as necessary. But the array has some memory space without use. Because the size of the array cannot be changed during the program execution, you must create an array big enough to store all the elements of the stack. However, normally you do not know in advance how many elements the stack will hold. So two things can happen. You either have unused memory space, or you run out of space if your stack grows bigger than the size of the array. This is the main drawback of using arrays to implement stacks or any other data structure that grows and shrinks on demand. Second, notice the stack has different rules for accessing its elements than those of the array or a linked list. In a stack the elements are inserted and removed from the same end. By contrast, in arrays and linked lists you can add or remove any element at any position. So this way of operating of a stack must be enforced when implementing it in an array or a linked list. Let's start by looking at the implementation of a stack using an array. When we create the stack, we create the array, and the variable top is equal to -1 to signal an empty stack. Now to push one element into the stack we need to perform two actions. First you must increase the value of the variable top by one unit. And second you must store the new element at the top of the stack. You can see that both actions take a constant time. So the time complexity of the push operation is Theta(1). Now because we are using a static array to implement the stack, it might be that after several executions of the push instruction we run out of space in the array. And therefore when executing the first instruction of the operation PUSH, we will get an execution error because we are beyond the limits of the array size. There are two ways of solving this problem. The first one blocks the execution of any further push operation once the limit of the array size has been reached. The second one uses the idea of dynamic arrays. A new array is created, usually with double the size of the original array. Elements are copied to this second array and the operation Push can proceed. In this second solution the time complexity of the push operation increases to Theta(n), because we need to copy all the elements already stored in the stack to the larger array. Now let's look at the POP operation. Remember that this operation removes the last element inserted into the stack, that is the element at the top. Some implementations of the pop operation return the value of the element being removed, and some don't. In this implementation we will assume that the value is not returned. We will use the operation peek for that. To perform the pop operation, we must simply decrease the value of the variable top. Notice that we do not delete the element from the array. By making the variable top one unit less than the position of the element deleted, we are not allowing any access to that element. For the function pop to take care of user misbehavior, we should also include an if statement in case the user wants to execute pop in an empty stack. So if the stack is empty, we should not allow further pop operations. You can see that each instruction inside the pop function takes a constant time. And therefore the time complexity of pop is Theta(1). Let's move to the next operation, PEEK, which returns the value of the element at the top of the stack. This operation is very simple, and it just returns the value at the top of the stack. All instructions inside the peak operation take a constant time to be executed, and therefore its time complexity is Theta(1). Finally, the operation ISEMPTY simply checks the value of top. If the variable top is equal to -1 then the stack is empty and the boolean variable TRUE is returned. In any other case the Boolean variable FALSE is returned. All the instructions executed by the ISEMPTY operation take a constant time, and therefore its time complexity is Theta(1). This table summarizes the time complexity of the stack operations when the stack is implemented with arrays. The only case in which a complexity is different from Theta(1) occurs when the array runs out of the space and must be enlarged to accept another element in the stack. Now let's move on to the implementation of a stack using a linked list. Let's start with an empty stack. In this case, top points to NULL. Now we execute a push operation by inserting the new element at the start of the list. By doing so, the content of top is changed from NULL to the address of the new element. If we execute another push operation, we insert another element at the beginning of the list. As a result, the value of top is updated to point to the new element. Here you can see the pseudocode of the PUSH operation for a stack implemented using a linked list. It's the same as the pseudocode of the operation insert at the start of the linked list. In this case, you do not have to worry about the linked list running out of space as it grows on demand. Because each instruction takes a constant time, the time complexity of the operation PUSH for a stack implemented with a linked list is Theta(1). Now let's look at the pop operation. Remember that this operation removes the last element inserted into the stack. That is the element pointed to by the top. This is the same as removing the first element of the list. So this is the pseudocode for the POP operation. First you check if the list is empty, in which case you cannot pop any element from it. And then you make the top bypass the element to be deleted by making it point to the second element of the list. We are assuming that because it's not pointed at by any other node, the first node of the list is automatically eliminated from memory. All instructions inside the POP operation take a constant time to be executed. And therefore POP has a time complexity equal to Theta(1). Now let's review the function PEEK. First we check if the list is empty, in which case we display a message and return. If the list has at least one element, then we return the data content of the first element. And because all instructions inside the PEEK function take a constant time to be executed, it's time complexity is Theta(1). Finally, let's look at the implementation of the operation ISEMPTY. This function is very simple. We simply check if top is pointing to NULL. if so, we return TRUE. Otherwise, we return FALSE. As with the previous functions all instructions take a constant time. And therefore the time complexity of ISEMPTY is also Theta(1). You can see that all stack operations have a time complexity equal to Theta(1) when implemented with a linked list. With this, we finish our study of how to implement a stack.

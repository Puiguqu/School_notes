This is the final linear data structure we will study in this topic. A queue data structure works in the same way a queue in real life works. People join the queue at its tail and once they are served, they leave the queue from the front of it. That is, a queue works in a FIFO manner, the first element in is the first element out. So this is how a queue is obstructed in computer science. The horizontal arrows show the direction in which elements move through the queue. Now, let's review the operations associated with a queue. The insertion of a new element to the queue is called enqueue. When you perform the enqueue operation, the element must join the queue by its tail. The removal of an element from the queue is called dequeue. When you perform a dequeue operation, you eliminate the element at the front of the queue. If you want to know the content of a queue, you can only look at the element at the front of it. So operations peek returns the value of the element at the front of the queue. Finally, the operation isempty returns the Boolean variable true if the queue is empty and false if it has at least one element. So a queue is a linear data structure with an important rule to follow. Elements are inserted at the one end and removed from the other end. A typical application of the queue data structure occurs when you have a server that can only serve one process at a time, but many processes can access the service. For example, a printer server, a web server, or an email server. In that case, the server can serve only one process at a time and the rest must wait in a queue.

The first of those will be a hierarchical data structure known as a tree. A tree data structures looks like this. The node at the top of the tree is the initial node of the tree, and it's called the root. In the same way a linked list has a pointer called head to point to the first element of the list, a tree has a pointer called root that points to the first element of the tree. In real trees, the root is at the bottom of the tree, so you can think of the tree data structure as an upside-down tree. ing this upside-down tree analogy, the nodes at the end of the tree are called leaves. And the arrows coming out from one node to another are sometimes called branches. The tree data structure also uses the analogy of a family tree to refer to some nodes. So the node at the beginning of an arrow is called a parent, and the node at the end of the arrow is called a child. For example, node B is the parent of nodes E and F, and nodes E and F are the children of node B. Additionally, all nodes on the way back from a node to the root are called the ancestors of that node. For example, nodes F, B, and A are all ancestors of node J. Similarly, all nodes on the way down from a node are called descendants. Finally, there are a couple of things that do not come from the analogy of an upside-down tree or from a family tree. The first is that every node in the tree has a depth, which is the number of branches between the root and the node itself. Thus, all children of the root have a depth of 1 because they are only one branch away from the root. The grandchildren of the root have a depth of 2 because they are two branches away from the root. Finally, the great-grandchildren of the root have a depth of 3. In bigger trees, of course there can be nodes with depths greater than 3. The nodes with the same depth belong to the same level of the tree, with the root belonging to level 0. The second thing is that the height of the tree is measured by the number of branches from the root to the deepest leaf. In this case, this tree has a height of 3. Depending on the problem you need to solve using a tree, every node in a generic tree can have as many children as necessary. Here, you can see that the root has three children, whereas other nodes have two, one, or no children at all. In other trees, the difference in the amount of children of different nodes could be even higher. When the number of children of every node in a tree is constrained to a maximum of two, that tree is known as a binary tree. This is an example of a binary tree. If every node has exactly two children except the ones in the last level, then we talk about a full binary tree. For example, this is a full binary tree of three levels. Since every node has a maximum of two children, binary trees are drawn in such a way that one child is drawn to the left of its parent and the other to the right. Because of that, they are called the left and the right child. For the same reason, we can talk about left and right sub-trees. For example, this is the left sub-tree of the root node. That is the tree rooted at the left child of the root node. And this is the right sub-tree of the root node. That is the tree rooted at the right child of the root. Inside a sub-tree, you can find other sub-trees. This might remind you of recursion. This characteristic of the trees, and in particular of the binary trees, is what makes them so good for applying recursive algorithms, as we will see later. With this, we finish our introduction to the tree data structure in general, and of the binary tree in particular. In the next lectures, we will focus on binary trees.

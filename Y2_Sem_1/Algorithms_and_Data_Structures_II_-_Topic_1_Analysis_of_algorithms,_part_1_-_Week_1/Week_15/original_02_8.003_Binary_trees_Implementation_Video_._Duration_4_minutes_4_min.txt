The first one uses memory pointers, as we did with linked list. The second one uses arrays. If we use memory pointers, we need to define a node with three memory spaces to store the following information; the data stored in the node, the memory address of the left child of the node, and the memory address of the right child of the node. For the sake of this example, I have also arbitrarily assigned this node the memory address 0xF. That is why the root, which is a pointer to a node, stores the memory address 0xF. This is an example of a note that stores number 5. As both the spaces for the memory address of its children are equal to null, this node is not yet connected to any other node. That is, this node does not have any children. If we were to draw the abstract representation of this tree, it would look like this. Sometimes, the fact that a node does not have a left, or right child, or both children, is made more explicit by adding this symbol. I usually will not show this symbol. If the node does not have a child, it will be obvious from the lack of an arrow. One, let's just start adding nodes to this tree. Let's create a new node that is going to be the left child of the root node. Let's assume that this node has number 13 stored in it, and that it has been created in memory address 0x23. As it is now, this new node is not connected to the tree. To connect it, we need to write its memory address in the space left of the root node. This is how the tree looks after we have done that. Let's now add a right child to the root node. To do so, we create a new node. Let's assume that node has number three stored in it, and that it has been created in memory address 0x17. Now, we connect this new node to the tree by writing its memory address in the space right of the root node. If we repeat this process, we can keep on adding nodes to our tree. So this is how we use memory pointers to build a tree. Another way of implementing a binary tree is using an array. In that case, the element at Level 0, that is the root node, is stored in position zero of the array. The elements at Level 1 are stored in positions one and two. The elements at Level 2, are stored in positions three, four, five, and six. Notice here that there is no node in position five. In that case, the array must store a number that is out of the range of the numbers stored in the tree. For example, if we know that the tree only stores positive numbers, then we cannot store negative one in the array to signal the lack of a node in that position of the binary tree. In general, elements at Level k are stored using 2_k positions, starting at position 2_k minus 1. The advantage of using an array to store a tree is that it uses less memory space since we do not have to use memory positions to store the memory addresses of the nodes. The drawback is that arrays are of fixed size, so you might be wasting memory space if your tree has a number of nodes lower than the array size, or you might run out of space if the tree grows larger than the maximum capacity of the array. During this topic, we implement trees using memory pointers.

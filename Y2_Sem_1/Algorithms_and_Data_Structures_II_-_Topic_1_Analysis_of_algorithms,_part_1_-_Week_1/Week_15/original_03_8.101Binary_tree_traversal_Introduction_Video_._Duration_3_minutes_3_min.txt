Traversing a tree is an important operation because in order to insert, delete, or search for a specific node, you must first be able to get to the correct position of that node. Traversal of the tree is the process of visiting all the nodes of a tree. Depending on the problem to solve, visiting might mean printing the content of the node on a screen, storing the content of the node in another variable, or checking if the content of the node meets some condition. Whatever the reason for visiting a node, when you traverse the tree, you must make sure you are visiting every single node. There are two main tree traversal approaches, breadth-first traversal and depth-first traversal. When you do breadth-first traversal, you traverse the tree horizontally, in the same way you read a page of text in the Western world, from top to bottom, from left to right. Thus, you first visit the root. Then, the nodes in level one from left to right. Then, the nodes in level two from left to right, and so on. In contrast, when you do depth-first traversal, you traverse the tree vertically, as if you were diving the tree. There are three different types of depth-first traversal; pre-order, in-order, and post-order. I have highlighted the prefix of each name in a different color because the prefix gives information regarding whether the root node is the first node to visit, the node to visit in the middle of the traversal, or the last node to visit. So, if you use pre-order traversal, the root node is the first node to visit. The prefix pre means before, so you visit the root node before any other node. After you have visited the root node, you visit the left sub-tree and then the right sub-tree. If you use in-order traversal, the root node is visited in the middle of the traversal. The prefix in means into. So, you first visit the left sub-tree, then the root node in the middle of the traversal, and finally the right sub-tree. Finally, if you use post-order traversal, the root node is the last node to visit. The prefix post means after. So, you visit the root node after having visited all other nodes. That is, you first visit all nodes of the left sub-tree, then all nodes of the right sub-tree, and then the root node. That is the overview of how to traverse a tree.

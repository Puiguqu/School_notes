A heap is a tree data structure like this. But for a tree to be a heap, it must satisfy two properties. The first one is known as the Heap Property. This property states what the value of the parent node must be in relation to the value of its children. If the heap is a max-heap, then the value stored in any parent node is always greater than the values stored in its children. In this example, the parent stores number 20, which is greater than the values 13 and 17 stored in its children. If the heap is a min-heap, then the value stored in any parent node is always less than the values are stored in its children. In this example, the parent stores number 13, which is less than the values 20 and 17 stored in its children. Let's have a look at this tree and check whether it satisfies the Heap Property. Let's start by looking at the root node and its children. The root node stores number 29, which is greater than the values stored in the children of the root, 15, and 20. So it seems that this tree might be a max-heap. Let's check that nodes at level 1 and their children also satisfy the Heap Property of a max-heap. The children of node 15 are nodes with the values 8 and 7. So the Heap Property of a max-heap is also met here. The next node at level 1, the node storing number 20, also has children with lesser values, 13, and 17. So we have checked that nodes at levels 0 and 1 satisfy the Heap Property of a max-heap. Let's move now to the nodes at level 2. We can see that the children of node 8 store the values 5 and 2, whereas the only child of 7 is 3. So at this last level the Heap property of a Max Heap is also satisfied. Therefore we can say that this tree satisfies the Heap property of a max-heap. Let's now discover the second property of a heap. The second property is called the Shape Property, and refers, naturally, to the shape of the tree. The Shape Property states that for a tree to be a heap, it must have a perfect triangle shape, or a triangle shape plus a left-aligned rectangle in the base. In tree terminology, the perfect triangle refers to a tree with all its levels full, and the rectangle refers to the last level that it can be not fully filled. But if that is the case, it must be filled from left to right. So this tree is a heap because it complies with both properties. Here you can see an example of a min-heap. It's a min-heap because first it satisfies the heap property. The value stored in every parent is less than the value stored in any of its children. The value 9 stored in the root node is less than the values 15 and 20 stored in its children. In turn, the values stored in the notes in level 1, 15 and 20, are less than the values stored in their respective children, 28 and 17 in the case of the node storing the value of 15, and 53 and 67 in the case of the node storing the value 20. Second, it satisfies the shape property by being a perfect triangle. So far I have shown heaps that are also binary trees. That is, heaps where every node has a maximum of two children. However, as long as the heap and shape properties are satisfied, any tree can be a heap. So we can have ternary heaps, that is heaps were nodes have a maximum of three children. Quaternary heaps with up to four children, or n-ary heaps, that is heaps with a maximum of n children, where n can be any integer number. However, given that binary heaps are the most commonly used heaps, in the rest of this topic we will work with binary heaps. So from now on every time I mention a heap, I mean a binary heap. With this we finish our introduction to heaps. Bear in mind that knowing the Heap and Shape properties is very important because they will affect the way the heap operations are implemented. So please keep them in mind for the rest of the lectures. For example, if you insert a new number in the heap, you must make sure that number is located in such a way that both properties are still satisfied. So You cannot insert any number in any position.

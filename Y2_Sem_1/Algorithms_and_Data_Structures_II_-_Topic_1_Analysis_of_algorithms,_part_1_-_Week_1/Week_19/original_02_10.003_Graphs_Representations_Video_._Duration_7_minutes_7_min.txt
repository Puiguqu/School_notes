Let's begin with the edge list representation and use this graph as an example. The edge list representation stores every link in the graph as a triplet; starting node, ending node, and the weight of the link. In the link highlighted here, that would be node J, node K, and number two. In the case of this graph, the edge list must have six elements because the graph has six different links. So if we stored this link in the edge list we'll store the triplet J, K, 2. Let us store the rest of the links. The edge between nodes K and M with weight equal to six, the edge between nodes L and J with weight equal to 10, the edge between nodes M and L with weight equal to eight, the edge between nodes M and N with weight equal to four, and the edge between nodes N and K with weight equal to one. In terms of memory consumption, this representation has a space complexity of Theta E, where E is the number of edges of the graph as you need to store E different triplets to store the whole graph. Now let's move to the next representation called an adjacency matrix. The adjacency matrix has the same number of rows and columns. This number is equal to the number of nodes of the graph. Here, outside of the matrix, you can see which row and column represent which node. The allocation of a node to a particular row or column is arbitrary but we usually adhere to the alphabetical or numerical order. Inside the matrix, there are as many numbers as the graph has edges. In this case, you can see six different numbers corresponding to the weight of the different links. If the graph is unweighted, the number 1 is normally used. Finally, a specific number is used to signal the absence of a link between two nodes. This is signaled here with the infinity symbol but the specific number to use depends on the numerical range used for the link weights. For example, if our graph is unweighted, then the presence of a link is signaled by using the number 1 and the absence of a link is signaled by using zero. If our graph is weighted and the weights are in the range 0-100, we may use number 101 to signal the absence of a link. The space complexity of using an adjacency matrix is Theta V_2, where V is the number of vertices of the graph. When the number of vertices in the graph is significantly higher than the number of edges, using an adjacency matrix might be inefficient in terms of memory space as most of the positions in the matrix are used to signal an absent link. In that case, you can use the third representation known as an adjacency list. There are two ways of building an adjacency list. In both cases, you create an array of lists. Every position in the array represents one node of the graph. In this case, position zero represents node J, position one represents node K, and so on. Initially, every position of the array points to null. Then to build the list, you look at the node represented by position zero, node J in this example. For every link going out from J, you add the information of the ending node and the weight of the edge. J has only one outgoing link, so you need to add only one element to the list of position zero. Then you move to the next node and follow the same procedure. After doing that with all nodes, you get this array of list. There is a second way of storing the same information shown on the right of the lower part of the screen where each node in the list stores the full information of an edge; starting node, ending node, and weight of the link. The space complexity of the adjacency list is Theta of V plus E, where V is the number of vertices and E the number of edges of the graph. Now before finishing this part, let's have a quick look at the most common graph operations. The first set of operations allows building a graph from scratch by creating the data structure and then adding vertices and edges. The second set allows deleting elements from the graph, either a node or an edge. The third and final set allows querying the graph. The first four operations allow the recovery of information regarding vertices, either the information about the whole set of nodes in the graph, the nodes at the start or end of a specific link or all nodes connected to a specific node. The last two operations allow the retrieval of information related to the links, either the information about the whole set of links of the graph or the weight of a particular link. This is the end of the representation of the graph as a data structure. As any data structure, the graph defines ways of organizing and manipulating data. In the case of a graph, you can select from three different ways of organizing its data: the edge list, the adjacency matrix, and the adjacency list. Additionally, you have a much larger set of operations than in the data structures we have seen before. They can be classified as operations to build the graph, to modify it, and to retrieve information. The way graph operations are implemented depends on the graph representation you select. In the next lectures, we will use these operations to build a few classical graph algorithms.

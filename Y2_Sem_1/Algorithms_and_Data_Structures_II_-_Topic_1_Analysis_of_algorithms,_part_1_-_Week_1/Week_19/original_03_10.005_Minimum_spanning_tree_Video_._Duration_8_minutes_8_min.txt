In this lecture, we will study two algorithms developed for a classical problem of graphs known as the minimum spanning tree. Let's begin by looking at one application of this problem and then study the algorithms. The men's football World Cup is one event watched by millions of people around the world simultaneously using streaming. If the last World Cup in 2018 had used a configuration like this with a single server streaming to the rest of the world, it wouldn't have worked. The server would have crushed due to too many data requests. Usually this problem is solved by using a network that has different servers interconnected across the world regions, and the streaming information is sent to them. Then users connect to their corresponding server to receive the streaming data. The key question here is, how is the information sent to the different servers? One way would be by establishing point-to-point connections from the server in different location to each of the different servers around the world. Here you can see a few of these point-to-point connections. Although this solution works, it has an important drawback. Exactly the same information is sent multiple times using the same communication link, leading to an inefficient use of bandwidth. To overcome that problem, a tree is calculated on top of the graph. The tree must comply with two conditions. First, it must include all the nodes of the graph. That's why it's called a spanning tree. Second, among all the possible spanning trees, it has to be the tree of minimum cost. That's why it's called the minimum spanning tree. The cost of the tree is usually calculated as the sum of the weights of its links. Using this tree, data is sent only once through each link, making efficient use of bandwidth resources. Now that you know at least one case where finding the minimum spanning tree is useful, let's look at the formal definition of the minimum spanning tree. The minimum spanning tree of a graph G is the tree that it includes all the nodes of G using the subset of edges with minimum total weight. In this example, the minimum spanning tree is shown by highlighting its links. You can see that this tree includes all nodes of key, and has a total cost equal to 10. You can try finding other spanning trees and check that this one is the minimum one. Also, any spanning tree of a graph with V nodes complies with having V nodes and exactly V minus 1 edges. To find the minimum spanning tree, there are two well-known algorithms called Prim's algorithm and Kruskal's algorithm. Let's review the idea behind each algorithm using this graph as an example. Let's start with Prim's algorithm. Prim's algorithm first selects any node in the graph to start building the tree. Let's say we arbitrarily select node A. So node A is the first node to belong to the minimum spanning tree. I have sign out this by circling A. As more nodes start to be added to the minimum spanning tree, this circle will enlarge to include them. , we identify all links that connect the nodes in the tree with the nodes not yet in the tree, and we select the link of minimum cost among them. This link as well as the node connected to it are included into the minimum spanning tree. The same procedure is now executed again. The links connecting the nodes in the minimum spanning tree to the nodes not yet in the tree are identified. The one with the minimum weight is selected and this link as well as the node connected to it are then added to the minimum spanning tree. As we continue repeating these steps, the minimum spanning tree is completed. So the next nodes to add are nodes E, F, H, G, D and finally I. In this way, we have built the minimum spanning tree using Prim's algorithm. Now, let's look at the idea behind Kruskal's algorithm using the same graph as an example. The approach of Kruskal's algorithm is to start with V different trees, each made of one node. Then these trees start to merge until they form a single spanning tree. The merging of the trees is done one step at a time, and at each step the minimum weight link connecting two different trees is chosen. In this case, the minimum weight link is the one connecting the trees made of the notes D and G. So these two trees are merged into a single tree made of nodes G and the link connecting them. Then the next link of minimum weight that connects two different trees is chosen. In this case, that is the link connecting the trees made of nodes A and B. So we connect those trees. The next link to add is the one connecting the tree containing node B and the tree containing node C. So these two trees are merged. The next link connecting two different trees is the edge of weight 4, connecting the tree containing node G and the tree containing node H. So we merge these two trees. The next step has two candidate links with the same weight, the link between nodes E and F and the link between notes D and H. However, only one of these links connects two different trees. So the link connecting the trees containing nodes E and F is selected and these trees are merged. At this moment, we have four different trees. So we still need to execute more steps as we need to have all the trees merged into one. The next link to select is the one connecting the trees containing nodes B and E. Finally, we select the last link to be part of the minimum spanning tree, the link connecting the trees containing nodes H and I, and we merge those trees. With this last merge, the minimum spanning tree is now completely built and both algorithms have found the correct solution to the problem.

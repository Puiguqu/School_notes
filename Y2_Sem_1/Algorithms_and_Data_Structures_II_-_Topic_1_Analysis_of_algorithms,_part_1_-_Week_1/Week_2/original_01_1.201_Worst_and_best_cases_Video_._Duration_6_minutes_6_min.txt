One example of an algorithm whose running time growth only changes with the input data size and not with the content of the data is find maximum. This algorithm returns the position of the element with the highest value in the array A. To do so, the algorithm uses the variable max to represent the index where the maximum value is. The algorithm starts by assuming that the maximum element is in position 0. Then it checks all remaining positions in the array using a for loop. If at some point during the loop, the algorithm finds an element that is greater than the element in position max, the value of max is updated. After checking all positions of the array, the value of max is returned. A quick analysis of this algorithm shows that the first line takes constant time. The for loop is a function of N. The conditional inside the loop is repeated N minus 1 times. So again, this is a function of N. And then, only one of the two return instructions is executed. And, in any case, they both take a constant time. So, the running time of this algorithm grows linearly with the size of the array A. In this case, no matter what numbers are stored in the array, the algorithm has to check every single position to decide where the largest number is. Thus, there's only a single case to analyze. One grows linearly with N, and this is the worst case because it represents the maximum number of instructions that the program must execute to find a result.

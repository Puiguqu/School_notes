This notation was created by a computer scientists in the seventies. By then, the analysis of the complexity of algorithms was performed by either using big O notation or Omega notation. One drawback of these two notations is that they refer to a set of functions. So if I say that the running time of an algorithm is big O n to the four, it might be that the algorithm grows with n to the square much faster than n to the four, but saying that it is big O n to the four is still correct. Theta notation finds a single function that acts simultaneously as an upper and a lower bound for our running time function or memory space function. So let's revisit this figure from the last two lectures. The function t of n represents the running time of an algorithm, the horizontal axis represents the input data size, and the vertical axis represents the time. In this example, the running time is proportional to n. With Theta notation, our aim is to find a function g of n that acts simultaneously as an upper and a lower bound for t of n, depending on the value of the constant it is multiplied by. So if g of n is multiplied by c_1, it acts as a lower bound for t of m. But if it's multiplied by c_2, c_2 is greater than c_1, it acts as an upper bound for tn. Let's look at an example. Assume that t of n equals 10 times n to the square plus 15 times n plus 5. We can see that if we choose g of n to be n to the square, then for big values of n, if g of n is multiplied by 1, it can act as a lower bound for t of n, and if it's multiplied by 30, it can act as an upper bound for t of n. Therefore, we can say that t of n is Theta, n to the square. If you try this with different functions, you will realize that the only function that complies with this condition is n to the square. Now lets define the Theta notation more formally. T of n is Theta of g of n. If the function g of n meets the following two conditions, g of n multiplied by a constant value c_1 is equal to or less than t of n for all values of n equal to or greater than n_0. G of n multiplied by a constant value c_2 is equal to or greater than t of n for all values of n equal to or greater than n_0. A more general way of expressing the same idea is as follows. A function f of n is Theta g of n, if there exist a positive constants, c_1, c_2, and n_0 such that c_1 times g of n is equal to or less than f of n, and c_2 times g of n is equal to or greater than f of n. Unlike Big O notation and Omega notation, which both correspond to a set of functions g of n, Theta notation only has one function, g of n.

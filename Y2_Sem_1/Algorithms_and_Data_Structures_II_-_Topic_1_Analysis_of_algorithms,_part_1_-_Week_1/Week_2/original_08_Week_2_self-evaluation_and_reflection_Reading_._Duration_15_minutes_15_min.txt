As you progress through your course, it is crucial to regularly assess your understanding and capabilities against the learning outcomes. This exercise is designed to help you reflect on your learning journey, identify areas where you may need to deepen your knowledge or skills, and develop a plan for improvement. Use this checklist to evaluate your understanding of the lessons covered in Week 2. If you don't feel confident about any of these concepts/tasks, revisit the relevant lecture videos, readings and activities to help you consolidate your knowledge. 1. Understanding Memory and Time Requirements I can analyse pseudocode to estimate the time complexity of an algorithm. I can analyse pseudocode to estimate the memory consumption of an algorithm. I can distinguish between different factors affecting time and memory requirements. 2. Growth Functions I understand what a growth function represents in relation to an algorithm's performance. I can determine the growth function for the running time of an algorithm. I can determine the growth function for the memory consumption of an algorithm. 3. Asymptotic Notation I can use **Big-O notation** to describe the upper bound of an algorithm's running time or memory consumption. I can use **Omega notation** to describe the lower bound of an algorithm's running time or memory consumption. I can use **Theta notation** to describe the tight bound of an algorithm's running time or memory consumption. I can distinguish the differences between Big-O, Omega, and Theta notations and when to use each. 4. Application and Analysis I can apply worst-case analysis to an algorithm and represent it using asymptotic notation. I can apply best-case analysis to an algorithm and represent it using asymptotic notation. I can compare different algorithms based on their time and memory efficiency using asymptotic analysis. 5. Reflection I feel confident in identifying and analysing the performance of an algorithm using pseudocode. I can effectively communicate the performance characteristics of an algorithm using asymptotic notation.

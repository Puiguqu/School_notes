The following resource provides a review of the key concepts covered in this module which encapsulates the foundational terms and concepts of algorithm analysis and data structures. You will need an understanding of these terms to complete your exam. General Terms Algorithm: A step-by-step procedure for solving a problem or performing a task. Pseudocode: A simplified, high-level description of an algorithm that uses plain language and programming structures. Time Complexity: A measure of the time an algorithm takes to run as a function of the input size. Space Complexity: A measure of the memory an algorithm requires as a function of the input size. Big-O Notation: Describes the upper bound of the time complexity in the worst-case scenario. Omega (Ω) Notation: Describes the lower bound of the time complexity. Theta (Θ) Notation: Describes the tight bound of the time complexity, encompassing both upper and lower bounds. Recursion: A programming technique where a function calls itself to solve a problem. Sorting Algorithms Bubble Sort: A simple sorting algorithm that repeatedly steps through the list, compares adjacent items, and swaps them if they are in the wrong order. Insertion Sort: A sorting algorithm that builds the sorted array one element at a time by picking elements and inserting them into their correct position. Selection Sort: A sorting algorithm that selects the smallest (or largest) element and moves it to the beginning (or end) of the list. Merge Sort: A divide-and-conquer sorting algorithm that divides the list into halves, sorts them, and merges them back together. Quicksort: A divide-and-conquer sorting algorithm that partitions the array around a pivot and recursively sorts the partitions. Counting Sort: A non-comparison-based sorting algorithm that counts occurrences of elements to determine their position in the sorted array. Radix Sort: A non-comparison sorting algorithm that processes digits of numbers in a specific order (e.g., least significant to most significant). Bucket Sort: A non-comparison sorting algorithm that distributes elements into buckets, sorts them individually, and combines them. Hashing and Hash Tables Hash Function: A function that maps data of arbitrary size to fixed-size values (hash codes). Collision: When two different inputs produce the same hash code. Collision Resolution: Techniques to handle collisions in a hash table, such as linear probing or chaining. Hash Table: A data structure that uses a hash function to map keys to values for efficient retrieval. Linear Data Structures Linked List: A data structure consisting of nodes, where each node contains data and a reference to the next node. Stack: A linear data structure that follows the Last-In-First-Out (LIFO) principle. Queue: A linear data structure that follows the First-In-First-Out (FIFO) principle. Sorted Linked List: A linked list where elements are maintained in sorted order. Trees Tree: A hierarchical data structure consisting of nodes connected by edges, with one root node and sub-nodes branching out. Binary Tree: A tree where each node has at most two children. Binary Tree (BST): A binary tree where the left child contains smaller values and the right child contains larger values. Tree Traversals: In-order: Visits the left subtree, root, and then the right subtree. Pre-order: Visits the root, left subtree, and then the right subtree. Post-order: Visits the left subtree, right subtree, and then the root. Heaps Heap: A specialized tree-based data structure that satisfies the heap property: Min-Heap: The parent node is less than or equal to its children. Max-Heap: The parent node is greater than or equal to its children. Heap Property: A condition that ensures the heap remains a valid structure. Heapsort: A comparison-based sorting algorithm that builds a heap from the input data and repeatedly extracts the maximum (or minimum) element. Graphs Graph: A data structure consisting of vertices (nodes) and edges (connections). Adjacency Matrix: A 2D array representation of a graph where each cell indicates whether an edge exists between two vertices. Adjacency List: A collection of lists where each list represents the neighbours of a vertex. Edge List: A list of all edges in a graph, with each edge represented by its two endpoints. Minimum Spanning Tree (MST): A subset of edges that connects all vertices in a graph with the minimum possible total edge weight. Prim’s Algorithm: An algorithm for finding the MST by growing the tree from an initial vertex. Dijkstra’s Algorithm: An algorithm for finding the shortest path between two nodes in a graph.

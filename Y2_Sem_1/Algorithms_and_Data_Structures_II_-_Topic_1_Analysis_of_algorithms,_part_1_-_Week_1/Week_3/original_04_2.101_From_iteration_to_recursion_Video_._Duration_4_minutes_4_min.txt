First, let's remember that the objective of both algorithms is repeatedly displaying or printing a number on a screen. So at some point they both execute a display or print action. The important issues are how the algorithms decide how many times to display information on a screen and how the information being displayed changes from time to time. With regard to the number of times the print or display instruction is executed both algorithms must determine an initial condition. In the case of the iterative algorithm that occurs when the control variable of the for loop is initialized. In the case of the recursive algorithm that of course by simply entering an input argument. , both algorithms must determine when to stop the repetition of code. That is when to stop printing on a screen. The iterative algorithm does so by verifying a condition. If the condition is true repetition of code continues. The recursive algorithm stops the repetition by using a base case where a condition is verified. If the condition is true, repetition of code stops. Notice that both the iterative and the recursive algorithms verify a condition. The conditioned state by the iterative solution must be true to continue the repetition. In contrast, the condition stated by the recursive solution must be true to stop the repetition. Finally in the iterative algorithm, the control variable of the loop must be updated to get closer to the value where the repetition stops. In the recursive algorithm, this is done by calling the recursive function with a different value in every repetition. In doing so, both algorithms update the number to be displayed on a screen. In summary, if you need to repeat code you can do so by iteration or recursion. Which one you choose depends on the problem you are trying to solve. In any case you need one or more initial values at the beginning of the repetition. You also need to define one or more condition values to either continue or stop the repetition. In the case of recursion the condition that stops the repetition is known as the base case. Finally, you need to update the values used in the condition to get closer to stop the repetition of code. Otherwise, you will face an infinite loop or infinite recursion. In most cases, recursive solutions can offer a more concise implementation than iterative solutions.

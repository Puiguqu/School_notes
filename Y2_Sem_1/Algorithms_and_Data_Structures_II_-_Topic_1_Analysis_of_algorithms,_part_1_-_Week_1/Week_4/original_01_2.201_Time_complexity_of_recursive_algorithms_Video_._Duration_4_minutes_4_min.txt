Now that you have practiced tracing and writing recursive algorithms, we will learn how to analyze recursive algorithms in terms of their running time. The time complexity of an algorithm is the asymptotic number of simple operations executed by the algorithm. In topic one, we learned how to calculate the time complexity of different non-recursive algorithms and how to express that using the Big O, omega or theta notation. Now, it's our turn to apply the same analysis to recursive algorithms. Let's do this using the recursive implementation of the algorithm that calculates the factorial of a number. Here you can see the recursive mathematical definition of the factorial function. For example, the factorial of three is equal to three times factorial of two. The factorial of two is equal to two times the factorial of one. The factorial is one is simply one. So the factorial of three is equal to six. Here you can see the algorithmic representation of eight using pseudo-code. This is the base case and this is the recursive call. Now, let's calculate the expression for the running time of this recursive algorithm. Let us start by defining t of n. The running time of the recursive algorithm that calculates the factorial of n in the worst case. Now, the first instruction takes a constant time to be executed. We will escape the return instruction as we are analyzing the worst case and then move to the line with the recursive call. In this line the return instruction, the recovery of the value of n and the multiplication all take constant time. But we don't know how long it takes calculating the factorial of n minus one. However, we know that t of n is the time required to calculate the factorial of n. Thus, we can say that the time required to calculate the factorial of n minus one is t of n minus one. If we define c one plus c two plus c three to be equal to c four, then the time taken by the line with the recursive call can be expressed as c four plus t of n minus one. Then we can say that t of n is equal to c zero plus c four plus t of n minus one. If we define c zero plus c four being equal to c five then we arrive to this expression for t of n. Notice how this expression is itself recursive. The running time of the algorithm tn is expressed in terms of the running time of the same algorithm, t of n minus one. This expression is known as a recurrence equation. A recurrence equation describes the running time of a problem of size n in terms of the running time of a smaller inputs. In the same way as a recursive algorithm delegates smaller tasks to the recursive call. Recurrence equations appear every time we need to calculate the running time of a recursive algorithm. Thus we need to learn how to solve them.

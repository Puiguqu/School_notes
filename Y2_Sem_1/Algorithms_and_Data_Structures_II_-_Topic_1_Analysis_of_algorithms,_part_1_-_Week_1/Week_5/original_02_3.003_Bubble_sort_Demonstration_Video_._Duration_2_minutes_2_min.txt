Let me start by telling you why it's called bubble sort. If you look at the bubbles in a fizzy drink, you can see how they rise to the top. In bubble sort, you will see exactly the same effect. If you are sorting the numbers from the smallest to largest, you will see how the highest numbers will go up in array. So if we put this five element array in a vertical position, as the algorithm gets executed, you will see how the highest numbers bubble up to the highest positions. We're going to start the execution of bubble sort now, with these five numbers, not yet sorted. So we move to the first position and we compared these two numbers, 31 and 12. Because we're sorting from smallest to largest, we are going to swap them. So the smallest number goes in this position. So after we swap, we move to the next position and compare these two numbers, three and 31. So we'll have to swap them. Now, we move to the next position, 23 and 31, we swap them. You can see how 31, that is the highest number here, is getting moved to the highest position of the array. Finally, we compare 31 and 17, and of course we swap them. By now, 31 is in its final position. We start the second pass now, we compare the first two elements, 12 and three, and we need to swap them. We move to the next position, 12 and 23, we don't need to swap. Then we move to the next position, 23 and 17, and we need to swap. You can see now that in the second pass, the second highest number has been moved to its right position. We don't need to compare these two numbers anymore because they are already sorted. We move back to the region, set pass three and 12, we don't need to swap them, 12 and 17, we don't need to swap them. Because we haven't done any swap, we know that the array is already sorted.

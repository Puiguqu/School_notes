There are a few different implementations of Bubble Sort. Maybe you have seen the classical one that is made of two nested for loops. The one we will study here is slightly different. Instead of one for loop nested in another for loop, it has one for loop nested in a while loop. As we will see, this implementation of Bubble Sort makes it fast in the best case, but in the worst case the time complexity is still theta n to the square. So you have a clear idea of what instructions belong to what loop or conditional statement. First, let's look at the while loop. This loop is in charge of detecting if at least one swap was made in the last pass through the array. If no swap was made, it means that the array is already sorted and then the algorithm finishes. If at least one swap was made, it means that the array might not yet be sorted and another path is necessary. To do this checking, the while loop uses the variable swapped. At the beginning of the algorith this variable is initialized to the Boolean value true, to make sure the algorithm starts the first iteration of the while loop. , immediately after entering the while loop, the variable swapped is made equal to false. That is, we are guessing that the array is already sorted. If so, the variable will not be changed and the algorithm will finish as soon as the condition of the while loop is checked. If during one pass through the array at least one swap if is made, then the variable swapped is changed back to true, meaning we need to execute the while loop once more. Let's now look at the nested for loop. This loop is in charge of visiting the elements not yet sorted by the algorithm. We call one entire execution of the for loop a pass since what it's doing is passing through the array. Besides visiting the elements of the array, the for loop is the core of the sorting process. Here is where a pair of consecutive elements is compared and if they are in the wrong order, they are swapped and the variable swapped is set to true. Notice that this pseudocode sorts the elements of the array from smallest to highest. Thus at the end of the first pass the highest value of the array is in its correct position. At the end of the second pass, the second highest value is in its correct position and so on. Because of that, every time we finish an entire execution of the for loop, we decrease the value of n as we do not need to check numbers already sorted.

You can visualize the structure of the algorithm as having a for loop with just two instructions inside it. The first instruction is a call to the function pos_ min and the second one calls the function swap. The FOR loop is in charge of determining the position where the next minimum number will be stored. In the first iteration the minimum number is stored in position 0. In the second iteration the second minimum is a stored in position 1 and so on. The function pos_ min returns the position of the minimum value between positions a and b, both inclusive, in the array A. For example, if we call the function pos_min with the input argument A, 2 and 4, pos_min will search the minimum value among the position 2, 3, and 4. In this case, the minimum value is number 3 which is in position 2. Thus pos_min returns the number 2, which is the position where the minimum value is found in the section of the array analyzed. The function swap exchanges the minimum value with the element occupying its correct final position.

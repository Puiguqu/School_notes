All you had to do was visit every position of array C and copy the corresponding number in R as many times as indicated. This leads us to the core steps performed by the counting sort algorithm. The input argument is an unsorted array A. The first step is to create the array with a count that is array C. In this case, you can see that array C stores the frequency at which the numbers in array A appear. The second and final step is creating the resulting sorted array R using the information that's stored in C. You can see that in creating arrays C and R not a single comparison was made. That is why this is a non-comparison sort. Let's now look at the same steps in detail using another small example so you can get a clearer idea of what is done at each step. This is the array A to be sorted. Then we'll start the execution of a step one. First, we create array C with the correct number of elements. Because the indices of array C represent the numbers stored in array A, C needs to have as many elements as the maximum value stored in A plus 1. , we count the frequency at which the elements in an array A appear and store that information in array C. To do so we visit every element in A and update the corresponding position in C. Notice that in counting half of the sorting task is done. If you look at the indices of array C highlighted in teal you can see that the original numbers in A are already sorted. Now we'll start step two filling in the content of the resulting array R with the sorted numbers. Array R has as many elements as array A. To store the correct information in R we visit every position in C and copy the corresponding elements as many times as required. We visit the first position of C but as it contains 0 we do nothing and proceed to the next position. Again, as there is nothing to do here we move to the next position. In position two there is a number 3 stored. Thus, we copy number 2 three times in the final array R starting at position 0. In the next position in array C position three we find number 1. So we copy number 3 in the next position of array R only once. In the next position of array C we find 0, so we don't need to do anything and therefore move to the next position. In the next position of array C position five we find number 2. Therefore, we copy number 5 twice into array R. In the next position there is 0, so we don't need to do anything. We find the same situation here. In the next position of array C we find number 1, so we copy number 8 once in the final array. Finally, in the next position of array C we find number 1, so we copy number 9 once in the final array. And with that the numbers are sorted in array R. Now that you are familiar with the idea of counting sort let's consider its two drawbacks. First, it can only sort integer numbers since array C uses the indices to represent the numbers to sort and the indices of an array are integer numbers. Second, C must have as many elements as the maximum number to sort plus 1. Therefore, the extra memory used by array C can be significant if the maximum number in the set of numbers to represent is too big.

# Summary & Skills Checklist - Anatomy of a JUCE application and macros
Reading•
. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-object-oriented-programming/supplement/7MbCf/summary-skills-checklist-anatomy-of-a-juce-application-and-macros

General Summary: In this lecture, we delved into the code generated by Projucer to understand how a JUCE application is structured and how it starts up. We explored the main components of the generated code, focusing on the use of macros and the initialisation process. We examined how the main function is generated and how the application is bootstrapped, leading to the instantiation of the main component where user-written code resides. Key Concepts and Examples: Exploring the Generated Code: Description: The lecture began with an exploration of the main.cpp file generated by Projucer, aiming to understand how the application starts and what code Projucer generates. Example: The absence of a traditional main function was noted, with the start of the application indicated by the START_JUCE_APPLICATION macro. Understanding Macros: Description: Macros are used to generate code during the pre-compilation phase, allowing for cross-platform compatibility by generating different code based on the target platform. Example: A simple macro was demonstrated, showing how it replaces code before compilation. The START_JUCE_APPLICATION macro was explored to understand its role in initialising the JUCE application. Examining the JUCE Macros: Description: The lecture traced the START_JUCE_APPLICATION macro to understand what code it generates and how it leads to the main function. Example: The macro was found to generate a main function and initialise the application using JUCE-specific initialisation functions. Initialising the Application: Description: The generated main function calls JUCE-specific functions to initialise the application and set up the main window. Example: The JUCEApplicationBase::createInstance function was identified as part of the initialisation process, leading to the instantiation of the main window and main component. Connecting to User Code: Description: The lecture explained how the generated code eventually connects to user-written code in the main component, where custom functionality is implemented. Example: The setContentOwned function was highlighted as the point where the main component is instantiated, transitioning control to user code. Checklist of Skills and Knowledge To successfully master the content delivered in this lecture, a learner must have the following skills and knowledge: Navigating Generated Code: Ability to navigate and understand the structure of code generated by Projucer. Familiarity with the main.cpp file and its role in the application. Understanding Macros: Knowledge of what macros are and how they work in C++. Ability to trace and understand complex macros like START_JUCE_APPLICATION. Initialisation Process: Understanding the initialisation process of a JUCE application. Familiarity with functions like JUCEApplicationBase::createInstance and their roles in setting up the application. Cross-Platform Compatibility: Knowledge of how macros aid in generating platform-specific code for cross-platform compatibility. Understanding the role of preprocessor directives in the compilation process. Connecting Generated Code to User Code: Ability to identify where the generated code transitions to user-written code. Familiarity with functions like setContentOwned and their roles in instantiating the main component. Exploring and Understanding Code: Encouragement to explore the generated code and understand its flow and structure. Knowledge of debugging techniques to trace code execution and understand macro expansions.

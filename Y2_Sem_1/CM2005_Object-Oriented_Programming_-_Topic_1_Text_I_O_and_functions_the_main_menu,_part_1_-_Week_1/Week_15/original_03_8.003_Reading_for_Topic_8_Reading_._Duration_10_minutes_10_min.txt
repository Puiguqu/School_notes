In this module, we are doing some refactoring. This leads to the concept of module coupling, which considers the level of interaction and integration between modules in a computer program. Here is a starter article about module coupling: 'Low coupling', Principles Wiki (updated 2014) https://web.archive.org/web/20191009061733/http://principles-wiki.net/principles:low_coupling accessed 17 April 2020. Another concept is module cohesion, which considers what is an appropriate set of functionality to put in a single module. Or, how cohesive that module is. This article is a good starter about module cohesion: 'High cohesion', Principles Wiki (updated 2019) https://web.archive.org/web/20191009061656/http://principles-wiki.net/principles:high_cohesion accessed 17 April 2020. We also see a pointer for the first time this week. We are not going to go fully into pointers in this course, we just use it as a convenient way to have an object shared between different bits of the program. If you want some heavy reading, you can refer to the textbook: Horton, I. and P. Van Weert Beginning CC++17: From novice to professional. (Berkeley, CA: Apress, 2018) 5th edition [9781484233665]. Chapter 6, p.177–181. Remember that all the Essential reading for this programme is provided for you. Click the link which will take you to the Online Library where you can search for the books.

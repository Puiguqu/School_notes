In this topic, the big area we're looking at is modules and how we can start thinking about how we design our software architectures in terms of their modules and how those modules interact. Let's have a quick look at the list of key areas that we're going to be looking at in the next four weeks. In the first week we're going to be introducing the concept of a module and explaining how we can think about model complexity because that's what underpins the rest of the work we'll be doing, with these topic set. In the second week, we are going to be introducing the concept of module cohesion. Now module cohesion is all about which things we put in a module and which things go into another module. For example, when you're designing a function, what should go into that function? At what point have you actually got two functions that you need to split apart? We're going to be thinking about how we can go about reasoning, about module cohesion. In the third week, we're going to be looking at module coupling. That's when you've got two modules and they need to interact with each other. We're going to be thinking about all the different ways that modules might interact with each other and think about when those interactions are desirable and when they're not desirable so we can better design the interactions between modules in our computer programs. In week four, we're going to be thinking about common programming concepts and language techniques and connecting those with the concepts that we've learned in the previous three weeks around module complexity, coupling and cohesion and having a really hands-on look at what specific techniques like variable scoping and function parameters and designs of data structures and how those impact upon the coupling and cohesion aspects of our software designs.

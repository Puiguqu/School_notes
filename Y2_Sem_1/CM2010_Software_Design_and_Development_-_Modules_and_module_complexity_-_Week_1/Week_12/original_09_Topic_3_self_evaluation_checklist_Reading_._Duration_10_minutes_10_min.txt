# Topic 3 self evaluation checklist
Reading•
. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-cm2010-software-design-and-development/supplement/Cr383/topic-3-self-evaluation-checklist

As you progress through your course, it is crucial to regularly assess your understanding and capabilities against the learning outcomes. This exercise is designed to help you reflect on your learning journey, identify areas where you may need to deepen your knowledge or skills, and develop a plan for improvement. Use this checklist to evaluate your understanding of the lessons covered in Topic 3. If you don't feel confident about any of these concepts/tasks, revisit the relevant lecture videos, readings and activities to help you consolidate your knowledge. Understanding robust and secure programming I can explain the principles of robust and secure programming. I understand the importance of defensive coding to prevent processing invalid data and ensure program stability. I can identify potential vulnerabilities and apply secure programming practices to address them. Assertion and parameter checking I can use assertions to verify assumptions about program behaviour during runtime. I can implement parameter checking to validate inputs and ensure correctness in program functions. I understand how assertion and parameter validation contribute to program robustness and security. Secure programming I can identify and mitigate common security risks in software development (e.g., input validation, buffer overflows). I understand how to prevent data leaks and secure sensitive information in my programs. I can apply secure coding principles to protect programs against common attacks, such as injection or unauthorised access. Exception handling I can use try-catch blocks to handle exceptions and errors gracefully. I can differentiate between checked and unchecked exceptions (where applicable) and handle them appropriately. I can design programs to recover from unexpected events and continue functioning without crashing. Using a debugger I can use a debugger to trace and identify errors or unexpected behaviours in a program. I understand how to set breakpoints, inspect variables, and step through code using debugging tools. I can analyse and resolve issues identified during debugging to improve program reliability. Writing programs I can write programs using variables, control flow, and functions while ensuring they are robust and secure. I can design and implement functions that handle invalid inputs and unexpected events effectively. I apply defensive coding practices consistently in all aspects of my programming. Practical application of robust and secure programming I can use exception handling and parameter validation to safeguard program execution. I can proactively identify areas of a program where defensive coding is required and implement appropriate solutions. I can balance program performance and security while maintaining readability and maintainability. Critical thinking and problem-solving I can evaluate the effectiveness of my defensive coding strategies in preventing program vulnerabilities. I can anticipate potential errors or security issues in code and design strategies to mitigate them. I understand the importance of robustness and security in professional software development and actively apply these principles in my work.

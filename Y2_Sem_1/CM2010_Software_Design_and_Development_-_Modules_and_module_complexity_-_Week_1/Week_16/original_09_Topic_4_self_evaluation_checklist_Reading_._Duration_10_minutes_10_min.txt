# Topic 4 self evaluation checklist
Reading•
. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-cm2010-software-design-and-development/supplement/ykevS/topic-4-self-evaluation-checklist

As you progress through your course, it is crucial to regularly assess your understanding and capabilities against the learning outcomes. This exercise is designed to help you reflect on your learning journey, identify areas where you may need to deepen your knowledge or skills, and develop a plan for improvement. Use this checklist to evaluate your understanding of the lessons covered in Topic 4. If you don't feel confident about any of these concepts/tasks, revisit the relevant lecture videos, readings and activities to help you consolidate your knowledge. Understanding user testing I can explain the purpose and importance of user testing in software development. I understand how user testing helps ensure that software meets user needs and expectations. I can describe how to carry out and evaluate user testing effectively. User stories I can write clear and actionable user stories that define what users should be able to do with a system. I can identify key user requirements and translate them into user stories. I can prioritise user stories to focus on the most critical functionalities for testing. Black box and white box testing I can explain the differences between black box and white box testing. I can design and perform black box tests to evaluate system functionality without knowing the internal code. I can design and perform white box tests to evaluate system behaviour based on code structure and logic. Usability testing I can design usability tests to evaluate how user-friendly a system is for its target audience. I can identify common usability issues, such as confusing interfaces or poor navigation. I can collect and analyse feedback from users to improve the overall user experience. Accessibility testing I understand the importance of accessibility testing to ensure software is usable by people with disabilities. I can evaluate a system’s compliance with accessibility standards, such as WCAG (Web Content Accessibility Guidelines). I can identify and address common accessibility issues, such as poor colour contrast, missing alternative text, or non-intuitive navigation. Practical application of user testing I can carry out user testing sessions and gather meaningful feedback from participants. I can evaluate the results of user testing and recommend improvements to the software. I can create a plan for iterative improvements based on user testing outcomes. Critical thinking and problem-solving I can identify the most appropriate testing methods (e.g., black box, white box, usability, accessibility) for different scenarios. I can analyse user testing feedback to distinguish between critical issues and minor suggestions. I understand the value of user testing in creating inclusive, user-friendly, and functional software.

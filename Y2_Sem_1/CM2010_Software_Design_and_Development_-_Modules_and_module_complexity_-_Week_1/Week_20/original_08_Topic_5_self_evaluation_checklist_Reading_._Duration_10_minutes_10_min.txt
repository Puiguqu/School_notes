# Topic 5 self evaluation checklist
Reading•
. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-cm2010-software-design-and-development/supplement/jFlRO/topic-5-self-evaluation-checklist

As you progress through your course, it is crucial to regularly assess your understanding and capabilities against the learning outcomes. This exercise is designed to help you reflect on your learning journey, identify areas where you may need to deepen your knowledge or skills, and develop a plan for improvement. Use this checklist to evaluate your understanding of the lessons covered in Topic 5. If you don't feel confident about any of these concepts/tasks, revisit the relevant lecture videos, readings and activities to help you consolidate your knowledge. Understanding version control I can explain the purpose and importance of version control in software development. I understand how version control systems help manage changes, track history, and collaborate effectively on projects. I can describe the basic workflow of a version control system, including creating, modifying, and managing repositories. Clone, create, add, and commit I can clone an existing repository and create a new repository in Git. I can stage changes using git add and commit them using git commit. I understand the importance of meaningful commit messages and can write descriptive messages for commits. Branch, merge, and conflicts I can create and switch between branches to manage feature development or bug fixes. I understand how to merge branches and resolve conflicts when merging changes. I can use tools to visualise branch histories and ensure a clean workflow. Using gitignore I can create and configure a .gitignore file to exclude unnecessary or sensitive files from version control. I understand the importance of not committing files such as configuration files, secrets, or build artifacts. Bug reporting and best practices I can create detailed and clear bug reports for public and private repositories. I understand how to use issue tracking systems to report and monitor bugs. I can follow best practices when reporting bugs, including providing reproduction steps, expected outcomes, and relevant logs or screenshots. Optimising your public GitHub profile I can create and maintain a well-organised public GitHub profile. I understand how to structure repositories with clear documentation (e.g., README files). I can showcase my work effectively by organising public projects and contributing to open-source repositories. Practical application of version control I can use Git to collaborate with others on shared repositories. I can resolve merge conflicts effectively and maintain a clean project history. I can manage branches for feature development, bug fixes, and release workflows. Critical thinking and problem-solving I can identify potential pitfalls in version control workflows and address them proactively. I can optimise my use of version control to improve collaboration, reduce errors, and streamline project development. I understand the impact of a well-maintained version control workflow on team productivity and project success.

Hybrid coupling, so what does it mean? Well, it's when you've got some data. So let's say we've got an integer value, which is, as we know, made out of zeros and ones, in its threads represented inside a machine. So that's my integer value as it's represented. The point is that Module 1 and Module 2 are going to use the same data, but in different ways or each module uses it in different ways depending on the state of it and things like that. So it's basically messy data use where you're saying, well, Module 1 is going to say, "Well, this is going to be used to represent the color of the screen." Then Module 2 might say, "Yeah. They might even split the data and say, "Well, you can have the first eight bits, I'll have second 8-bits and we'll store in the same place." That's an example of hybrid coupling. As you might imagine, that's generally bad. However, I'm just going to illustrate a scenario where actually hybrid coupling might be your only option. So if you're in a situation where you're on some microcontroller that's got very limited resources, you might find that you have to make multiple uses of blocks of memory.

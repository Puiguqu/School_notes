* Test-driven development (TDD) is an agile software development methodology that involves writing automated tests before writing the actual code.
* Unit-testing is a type of testing that focuses on individual units of code, such as functions or methods, to ensure they work correctly and meet specific requirements.
* The three laws of unit-testing are principles that guide the development of unit tests:
 1. **Write tests before writing code**: This principle emphasizes the importance of writing tests before starting to write code.
 2. **Keep tests simple**: This principle suggests that unit tests should be straightforward and easy to understand, with a clear focus on testing specific functionality.
 3. **Automate tests**: This principle advocates for automating as much of the testing process as possible, using tools like unit test frameworks and continuous integration pipelines.

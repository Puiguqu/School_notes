# Topic 2 self evaluation checklist

. Duration: 10 minutes

URL: https://www.coursera.org/learn/uol-cm2010-software-design-and-development/supplement/VNXDu/topic-2-self-evaluation-checklist

As you progress through your course, it is crucial to regularly assess your understanding and capabilities against the learning outcomes. This exercise is designed to help you reflect on your learning journey, identify areas where you may need to deepen your knowledge or skills, and develop a plan for improvement. Use this checklist to evaluate your understanding of the lessons covered in Topic 2. If you don't feel confident about any of these concepts/tasks, revisit the relevant lecture videos, readings and activities to help you consolidate your knowledge. Understanding test-driven development I can define test-driven development (TDD) and explain its principles. I understand the 'red-green-refacto' cycle and how it guides software development. I can explain the advantages and challenges of using TDD in software development. Unit testing in Python I can write unit tests using Python’s unittest framework. I understand how to use assertions in unittest (e.g., assertEqual, assertTrue, assertFalse). I can create and execute test cases for Python functions and modules. Unit testing in C++ I can write unit tests using a C++ testing framework. I understand how to use assertions in C++ unit tests. I can create and execute test cases for C++ functions and classes. Unit testing a web API I can write unit tests for web API endpoints to validate their functionality. I can test HTTP response codes, payloads, and behaviours of RESTful API methods (e.g., GET, POST). I understand how to use tools or frameworks for API testing, such as Postman or pytest with requests in Python. Writing programs I can write programs in Python and C++ using variables, control flow, and functions. I can design functions that are modular, reusable, and testable. I understand how to apply good coding practices to create reliable, maintainable code. Practical application of TDD I can write test cases before implementing code as per TDD principles. I can iteratively write, test, and refactor code to ensure quality and reliability. I can evaluate and handle errors in failed tests effectively. Critical thinking and problem-solving I can identify and write test cases for edge cases and potential failure points in code. I can evaluate the trade-offs between using TDD and other development methodologies. I understand the role of unit testing in ensuring robust software design and implementation.

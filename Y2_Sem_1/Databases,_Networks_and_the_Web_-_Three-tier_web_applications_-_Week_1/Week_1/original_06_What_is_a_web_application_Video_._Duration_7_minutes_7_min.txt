Dynamic web applications in contrast provide more user interaction. The user can input, change, and manipulate data on their website. Web application send a lot of request to a web server in order to get a response. The most obvious examples of dynamic web apps are social networks and online editors such as quora.com. You might be wondering what the differences are between a web application and a traditional desktop application. You can have a look at a good explanation of this under techterms computer dictionary, which I will give you a link to as further reading for this topic. Briefly, the key difference is that a web app is a software program that runs on a web server and is accessed through a web browser, whereas traditional desktop applications are launched by your computer's operating system. There are advantages and disadvantages to each. From a user's point of view, with web apps, they can access the same data from multiple devices and with the same experience, rather than transferring files between different computer systems that work in different ways. On the other hand, if the browser crashes, you may loose your unsaved data, and it's slower in general because it has only limited access to your system resources such as the computer's file system and memory.

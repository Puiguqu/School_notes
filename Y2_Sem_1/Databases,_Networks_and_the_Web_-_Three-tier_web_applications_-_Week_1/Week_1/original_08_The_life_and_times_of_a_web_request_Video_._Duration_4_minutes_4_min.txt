In this video I'm going to start with the question we ended with. What happens when you call a web application? As I said then everything starts by typing a URL in a web browser. You probably do that all the time, but what actually happens when you type a URL into a browser? Let's have a closer look. When a URL is typed in the browser, the browser cache is checked as you can see in step two. Then the domain name server or DNS, which is the Internet's equivalent of a phone book or a contact list, performs a lookup to find the IP address of the server. Domain name servers maintain a directory of domain names and translate them to Internet Protocol or IP addresses. Each IP identifies a machine connected to the Internet. Do you know what your IP address is? then a step for the browser initiates a TCP connection with the server. TCP stands for transmission control protocol. This facilitates communication between a machine and the Internet. Then in the next step the browser sends an HTTP request to the server. HTTP or hypertext transfer protocol is the main protocol used to transfer hypertext pages which are written in HTML from a web server to a browser. The server then handles the incoming request and then the browser receives the HTTP response. Finally the browser displays the HTML content and you can see the web applications you wanted to visit. You will learn more about network protocols later. For now let me explain a little more about HTTP protocol. In an HTTP protocol, we have a request and a response. The client, which could be the web browser, makes a request for a particular resource. Scripts on the server evaluates the request and return a response which may or may not include the requested resource plus some other stuff. The web browser will evaluate the response and decide what to display and how to display it. Going back to a step seven and eight in the life and times of the web request. What if it's a dynamic web application and not a static website? Where will the page content come from then? Dynamic websites are not based on files in your computer's file system rather they are based on programs which are run by the web server when a request comes in and which generate the content that is returned to the user. One major source of such content is a database. We won't worry about the details of this just yet.

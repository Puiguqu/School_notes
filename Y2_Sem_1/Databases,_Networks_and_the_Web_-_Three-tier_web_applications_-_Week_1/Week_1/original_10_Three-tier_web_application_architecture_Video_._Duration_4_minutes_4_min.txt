Three-tier architecture is a client server architecture in which the user interface, application logic and data storage are developed, and maintained as independent modules. A tier in this case is a layer. The presentation or user interface tier receives input and displays output. Then the application tier processes the logic and makes the calculations. Finally, the database tier stores and manages data. As an example, think about an e-commerce web application. The presentation tier would include web browsers running on desktop computers, tablets or mobile devices. The application tier would be all the code required to do the necessary calculations such as adding up the value of goods in the shopping basket and applying discount vouchers and so on. The data tier would be all the files and database management systems that contain the data for shipping and order processing. There is a really helpful explanation of this in designing data intensive web applications by Stefano Ceri et al. They say the core principle of three-tier architecture is to provide an intermediate layer between the client and the data tier which centralizes middleware services and the business logic of the application. Three-tier architectures were proposed in alternative to two-tier architectures, in which client programs interact directly with database management systems by posing queries, receiving the results, and processing them for presentation to the user. Three-tier architectures offer a higher degree of scalability than two-tier configurations, thanks to better network utilization and to the virtually unlimited replication and load distribution capabilities of the middle tier. Their terminology may vary a little, but essentially it means the same thing. The presentation tier is sometimes called the front end. The application tier is referred to as the middleware and the database tier is called the back end. You've probably also heard people talking about client and server sides of web applications. So how does this fit in? Which tiers are on the client side and which are on the server side. Well the front end or presentation tier is on the client side, and the application and data tiers or middleware and back end are on their server side. It's useful to know some of the different languages that are executed on the client and server sides. Some examples of client side languages are HTML, CSS and JavaScript. In the middle web part of the server side, you can have JavaScript, PHP, Python, Ruby and so on. One of the most famous languages in the data tier of server side is SQL.

In the concept of relational databases, a primary key is a field or combination of fields in a table that uniquely identifies each row or a record in a table. The answer to this question is that in most of the cases, primary key is a numerical ID. A primary key is often an integer ID field set to auto increment. Now, when we are designing databases, we need to make sure that we maintained accuracy and integrity of information within them. There are a number of rules which help us to do that. Constraint as it is understood and they're in database design is a rule imposed by the database management system that cannot be broken by the users of the database. Examples of constraints are limiting acceptable values and actions such as not null constraint. Some integrity constraints are related to primary keys, or PKs for short. We call them the primary key constraints.

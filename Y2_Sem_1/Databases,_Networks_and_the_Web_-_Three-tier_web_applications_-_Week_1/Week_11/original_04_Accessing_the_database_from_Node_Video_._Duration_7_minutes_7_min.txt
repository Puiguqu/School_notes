It should connect to your database, retrieve data from the database and show it in a web page. Let's take a little time to understand how this code is working. The first thing you did was install a Node.js module called mysql2 using npm install mysql2. This installed a library of functions that allow you to perform database operations directly from within your Node.js application. you imported the library, adding a require command in index.js. you connected your Node.js application to your database. For this, you used the createConnection function of MySQL. You can imagine that there are many different MySQL databases in the world, running on many different computers. So the first thing we need to specify is what computer the database is running on. This is called the host computer, in other words, the computer that is hosting the database. This might be the IP address of a computer running MySQL, or the network name of a computer running MySQL. For your application, MySQL is running on the same computer as your Node.js program, so you can use a special name called localhost that connects back to the same computer that your application is running on. Each MySQL installation can have multiple users set up, so MySQL needs to authenticate a valid user in order to allow access to the database. The root user is present on all MySQL installations and has access to all the databases. So we can use that, specifying the username root and the password that you specified when setting up MySQL. Now you might be concerned about accessing the database using such a powerful user, and you'd be right to worry. We will look into this more when we examine security in a later topic. Because there are multiple databases on the MySQL installation, we also need to specify which database we want to access. As this is our bookshop app, we will specify the mybookshop database. Having defined the connection, we can then request a connection to the database, where we use the connect function to achieve this. Note that there are many reasons why this connection could fail. If the host, user, password or database are incorrect, an error will result. And because we are dealing with network connections, a network error could also result in a failure. Or perhaps even the database server may be offline. So it is a good idea to add some error handling to report an error in case of some failure. In this case, we are just throwing the error, so the application will terminate. Now that the application is connected to the database, you can start performing database operations directly from within your application. Let's take a look at the code you wrote to show the list books page. First let's think about what we want our application to do when a user browses to the list page. The request will be sent to our application tier, which is the Node.js application. The application tier will need to contact the database to request the data required. This request will take the form of a SQL statement, select star from books. The database must then return the data back to the application tier. And the application tier will then need to populate the list page with a list of books. Let's take a look at the code required to achieve this. The application tier code is in the slash list root handler in main.js. List.ejs contains the EJS code that will receive and display the data. The slash list root handler is called in response to a user accessing the slash list root. The database query select star from books is set up. And the query is executed by calling db.query. Note that this is an asynchronous function, so the query is kicked off and the application continues running. This anonymous function is called when the query completes. At this point, the query may have succeeded or it may have failed. The err parameter to the function will be set if there was an error. In the case that there is an error, the code passes the error to the next matching root handler middleware function. In other words, delegating the handling of the error to the next handler. Because we don't have another handler for the slash list root, this effectively asks express to deal with the error, which it will do by showing the error on the web page. If there is no error, the list.ejs page is rendered using the result of the query. Note that the result is an array representing the data stored in the books table. Note also that the data passed to list.ejs is an object containing a variable called available books that is set to the value of result. So when we come to populate the EGS page, we need to pull the data from the available books variable. The loop in the EGS page loops through the list of results returned from the database and extracts the name and price from each item in the array.

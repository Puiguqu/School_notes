Here, we are talking about event-driven programming compared to sequential programming. Let's just refresh our memories with what we learned about events earlier in this course, as that helps us understand event-driven programming. Event-driven programming is different from sequential programming in the sense that events such as user interaction and sensor outputs control the flow of the program rather than relying on a preset sequence. As you will remember, user interactions include events such as mouse clicks and keyboard strokes. As a developer, when you are writing an event-driven program, you need to think about possible events and how the program is going to respond to each type. In the case of web application development, when the user types a URL into a web browser, that is an event. You respond to it by calling the callback function written for that specific route. In sequential programming, on the other hand, the flow of the program is only controlled by the sequence of statements. There is a start statement and an end statement that clearly define where the flow of the program begins and ends. And in between, there are statements which run one after the other as implemented in the software program. Again, as I said before, the views folder hosts HTML and EJS files.

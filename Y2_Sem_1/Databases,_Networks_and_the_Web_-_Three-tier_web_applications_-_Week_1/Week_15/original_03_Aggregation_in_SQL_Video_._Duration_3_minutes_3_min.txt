Aggregation is a powerful feature of relational database operations. Let's take a little time to understand how these aggregation functions are working. When we run the query SELECT COUNTID FROM BOOK, we are counting the number of non-null IDs in the book table. In this case, there are 7. When we run the query SELECT SUMPRICE FROM BOOK, we are asking for the sum of all the price values from the book table. And the answer in this case is 209.12. We use the GROUP BY clause to group the results of an aggregation function by another column. In this case, we are asking for the count of IDs from the book table, but grouped by the CATEGORY column. So MySQL will take the rows where the category is Computing and count these. And take the rows where the category is Science and count these. And the result is 3 Computing and 4 Science books. Things get a bit more complex when we want to aggregate across multiple tables. First, we need to join the tables together. In this case, joining the book table to the publisher table on the publisher ID. Effectively, this gives us a combined books and publishers results set, as shown in the table here. Then, we group by the column requested, publisher name. Now all the Avon books, CAM books and Thames books rows have been grouped together, indicated by the different colours here. We then select the publisher name and count the rows in each publisher name grouping. And this gives us this result with 2 Avon books, 3 Thames books and 2 CAMs books. You can also group by multiple columns. In this case, we want to group first by publisher name and then by book category. Here, to make things clearer, I've reordered the rows by the publisher, indicated by the different colours. And then within each publisher group, we can count the book categories. For example, in blue you can see that Thames books has 1 Computing book and 2 Science books. As you can see, these queries can get quite complex.

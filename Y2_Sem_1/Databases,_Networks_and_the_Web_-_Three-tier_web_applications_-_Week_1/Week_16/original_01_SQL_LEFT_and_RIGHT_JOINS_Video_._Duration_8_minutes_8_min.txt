As you may remember, when retrieving records from multiple tables related to each other by a foreign key, SQL SELECT join can be used. To do this, we start with the keyboard SELECT and then the name of the fields we want to retrieve. Now let's write a specific SQL query to retrieve the name of an artwork and its artist, in addition to the name of the museum displaying that artwork. The SQL join statement would be SELECT Artwork.name, Artwork.artist, Museum.name FROM Artwork JOIN Museum ON Artwork.museum_id = Museum.id. The condition after the ON keyboard in an SQL SELECT join statement will specify the condition for joining the two tables based on the name of the foreign key and the name of the referencing primary key. The condition after ON will retrieve the records in which the value of the specified foreign key matches the value of the primary key in the referencing table. In our example, SQL join will merge the data in the artwork and museum tables based on the values of the foreign key museum ID and the referencing primary key ID in the museum table. It will then list data retrieved from the two tables. Running this SELECT statement in MySQL Shell will result in a list of two records, Color madness, Junior Picasso, Art Museum, and then Beautiful lake, Luke Real, National Museum. We consider one table, table A, as the set of records stored within it and table B as the second set called B storing data records of table B. Therefore, we can say that SQL SELECT join finds records that belong to both tables, or we say that they are in the intersection of set A and set B. INNER JOIN is another name for JOIN. An INNER JOIN includes only rows with matches in both tables. So as we said at the start of this lecture, there are two other forms of join, left and right joins. Left or right joins include all the rows from one table and any matching rows from the other table. Let me explain them one by one with our museum database example. Left join includes all the rows from the table on the left and any matching rows from the other table. For example, SELECT Artwork.name, Artwork.artist, Museum.name FROM Artwork LEFT JOIN Museum ON Artwork.museum_id = Museum.id will return all three records in the artwork table with associated museum name. The museum named for the first artwork will be displayed as null because this artwork is not on display in any museum. Right join, on the other hand, will retrieve all the rows from the right-hand table and any matching rows from the other table. As an example, SELECT Artwork.name, Artwork.artist, Museum.name FROM Artwork RIGHT JOIN Museum ON Artwork.museum_id = Museum.id will return the two records in the artwork table that are displayed in a museum. In addition to the data related to the third museum with no artwork, namely modern museum. The name and artist of the third item will be displayed as null. There are other forms of SQL joins, such as left excluding join, right excluding join, outer including join, and outer excluding join.

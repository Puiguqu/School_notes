You may have noticed that one of the artworks was not linked to a museum, and one of the museums was not linked to an artwork. When we run a SQL query that joins the two tables on the primary foreign key relationship, some artworks and museums are connected together, some artworks remain unconnected, and some museums remain unconnected. We can think of these two tables as being sets. One set is a set of artworks, the other set is a set of museums. Some museums and artworks are connected by the primary foreign key relationship, and these are shown in the intersection of the sets. But some data on the left and right tables are not connected, and these are shown outside of the intersection of the two sets. When we perform an inner join using the join keyword on its own, only the intersection of the sets is considered, shown here as a grey shaded area. The result set will exclude artworks and museums that are not connected together, showing only the two rows represented by the intersection of the sets. When we perform a left join using the left join syntax, we include all rows from the left hand table, and only the matching rows from the right hand table. This selection is shown in the grey shaded area here. The result set will include all three artworks, but where the artwork has no matching museum, the museum columns will be set to null. The museum that has no artwork in our data set, the Louvre, is not shown. When we perform a right join using the right join syntax, we include all rows from the right hand table, and only the matching rows from the left hand table. This selection is shown in the grey shaded area here. The result set will include all three museums, but where the museum has no matching artwork, the artwork columns will be set to null.

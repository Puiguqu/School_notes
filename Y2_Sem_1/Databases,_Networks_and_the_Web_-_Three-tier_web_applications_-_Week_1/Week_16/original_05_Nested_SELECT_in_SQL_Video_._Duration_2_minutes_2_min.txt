Let's take a look and review what you did. In the lab on NestedSelects, you were given a query that found all books that are more expensive than the book WebArchitecture. The query utilised a NestedSelect to achieve this. The inner query, SELECT PRICE FROM book WHERE name == WebArchitecture, finds the price of the WebArchitecture book, which returns $31.99. The outer query, SELECT NAME, PRICE FROM book WHERE price greater than, finds all books that are priced more than the result of the inner query. So any books priced more than $31.99 are selected. In the lab, you were given this additional exercise to find books that are cheaper than the cheapest computing book. How did you get on? With these sorts of problems, as with a lot of problems in computing, it always helps to break the problem down. First, find all the computing books. To do this, a query SELECT STAR FROM book WHERE category == Computing will do the job. Then we can refine this to find the minimum price in the computing books. For this, we can add an aggregation minPrice. Given this minimum price, you can select other books that are cheaper than it. We nest the query that selects the minimum price for a computing book inside another query that selects books that are cheaper than it. The query shown in blue here, SELECT NAME, price, category FROM book WHERE price less than, is what we need.

When storing the password, you should have encrypted the password to keep it secure. Let's take a little time to understand how this code is working. When a user fills in the register form and presses the OK button, a POST request is made to the application. And we would expect the route handler to insert the data from the form into a users table. But if we insert the raw password into the database, there is a risk that a malicious person who gains access to the database would be able to use the password to gain access to our application. Or potentially to other applications if the user uses the same password across different systems. So rather than storing the raw password, we should store an encrypted password. Let's take a look at the middleware function that achieves this. The raw or plain password is extracted from the request body. This is the raw password that was entered by the user in the registration form. The raw password is passed to the bcrypt hash function, together with the number of salt rounds we want. The more salt rounds we specify, the longer the hashing operation takes. Remember that slowness is good because it slows down a potential brute force attack. But we don't want it so slow that the user gets impatient. As an experiment, you could try to increase this value little by little, and you will notice that the registration process gets noticeably slower. The bcrypt hash function is asynchronous, so you will need to provide a callback function that is run when the hash operation is complete. Notice that the callback function has two parameters, an error and the hash password. When bcrypt calls your callback function, it will provide the hash password here. The callback function sets up a SQL INSERT statement to insert the form details into the users table. Notice that the last placeholder is for the hashed password rather than the plain password, because we want to insert the hashed password into the database. So we will extract all the data from the form except for the password. Instead of the plain password, we will use the hashed password generated by the bcrypt hash function. Finally, we can run the query to insert the form data and hashed password into the database. If there is a database error, this is passed to the next middleware function.

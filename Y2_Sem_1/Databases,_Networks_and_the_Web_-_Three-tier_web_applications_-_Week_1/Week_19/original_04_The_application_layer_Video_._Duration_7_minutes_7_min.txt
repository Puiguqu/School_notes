The application layer is the layer in which you as a computer science student and later graduate will most probably work developing dynamic web applications as well as other software programs. It is the highest layer in the model closest to the user and provides interfaces and protocols that users need. The application layer deals with the highest level protocols that handle communication between computers. This is protocol such as HTTP, FTP, SMTP, SSH, DNS, DHCP telnet and many more. As you can guess from the protocol names, the application layer provides services such as browsing, email, file transfer, chatting, directory and so on. The application layer protocol defines the types of messages that are exchanged, in other words, whether there are requests or responses. It also sets the message syntax. In other words, what fields are included in the messages and how those fields are delineated and the message semantics which is the meaning of the information in those fields. If you are interested to see what a protocol looks like, you can have a look at RFC 2616 to see the HTTP 1.1 protocol as the foundation of data communication on the web. This is an example of HTTP request and response. It is a client request by the get method and the server response includes all the details of the server last modified time content type and much more. Let me give you more more information about two application protocols. Since I have already introduced HTTP to you earlier in this course, let me talk about HTTPS and FTPS, as extensions to the commonly used HTTP and FTP respectively. Online security is an important and necessary part of web application development nowadays. Users of a web application will only trust it if they are assured that a secure connection is established. SSL or secure socket layer assures users that the connection is secure. Modern browsers may warn users if a site that ask for data doesn't have a valid SSL certificate. HTTPS actually stands for HTTP over SSL. So, HTTPS and FTPS use SSL to encrypt data sent over a network. SSL establishes and encrypted link between a client and a server to provide security. There are a number of issues to consider when providing online security including data in motion encryption, client and server authentication, and data integrity mechanisms. SSL deals with all of this. Built-in motion encryption encrypts the data to be transferred to keep it confidential and to make sure it will not be readable by anyone other than the recipient. Client and server authentication makes sure that the client and server really are who they claim to be. Data integrity mechanisms make sure that any sensitive data is received as it was originally sent by the sender. I've said up to now that HTTPS is HTTP over SSL and that used to be the case. However, TLS which is stands for transport layer security is the modern encryption standard. SSL and TLS are both network protocols that allow data to be transferred privately and securely between a web server and web browser. SSL is older but it still works well, it's important to note that sometimes when we are talking about HTTPS it's really TLS rather than SSL. This diagram shows the link, network and transport layers, and above that application layer. In the application layer you can see HTTP alongside the various different parts of the SSL protocol. The SSL record layer protocol. SSL handshake protocol, SSL change cipher spec protocol, and SSL alert protocol, all together this makes up the HTTPS protocol. A similar principle applies with TLS. These different parts of the SSL protocol are not the subject of this course, so I'm not going to go into any more detail about them now. Browse through the course reading materials if you are interested to learn more about this area. So in summary, HTTPS is the HTTP protocol embedded within the TLS protocol or the SSL protocol. HTTP is responsible for handling requests and responses and all the web serving mechanics. And TLS or SSL are responsible for encrypting the data sent over the network and for authenticating the identity of the server host using a certificate. The case of FTP is similar to HTTP, in the past there was only FTP, but now to protect file transfer from attacks FTPS is used. FTPS also known as ftps FTPS, FTP SSL, and FTP secure is an extension to the commonly used FTP adding support for the TLS and formerly the SSF. In this lecture, I have introduced you to the application layer, which is the highest layer in the TCP IP model.

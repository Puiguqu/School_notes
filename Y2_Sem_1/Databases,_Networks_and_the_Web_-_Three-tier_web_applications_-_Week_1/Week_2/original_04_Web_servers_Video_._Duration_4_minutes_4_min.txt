But what is a web server? As we already know from learning about three tier web architecture in the previous topic, when you are creating a web application for serving data or information. You need a web server to be constantly running and handling requests from users. In more technical terms, TechTarget defines a web server as a program that uses HTTP or Hypertext Transfer Protocol. To serve the files that form web pages to users in response to the request, which are forwarded by their computers HTTP clients. They also note that dedicated computers and appliances may be referred to as Web servers as well. As always, it's useful to look at definitions from more than one source to get a good understanding. The Wikipedia entry on web servers notes that a web server is server software or hardware dedicated to running set software, that can satisfy World Wide Web client requests. A web server can, in general contain one or more websites. A web server processes incoming network requests over HTTP on several other related protocols. So those are some fairly good definitions off a web server. But what about the other term that we have been using a lot, HTTP? This is really well defined in one of your readings. Page five of Designing Data Intensive Web Applications by Stefano Ceri. You might remember this book from Topic one. These authors tell us that the fundamental technology at the origin of web applications is the well known Hypertext Transfer Protocol. Technically speaking, HTTP is a client-server application protocol, which defines the rules by which a client program called a browser or user agent on a server program called a Web server may in tracked in order to exchange request on responses. In HTTP terminology, the user agent sends a request from a given resource to the Web server. Which is a process running continuously and listening to request coming from the network. Upon receiving the request, the server locates or builds the resource and sends a response to the client. The basic resource requested by the client is an HTML page, which is a piece of text representing a multimedia hyper textural document. More generally, a request may address a file off any format stored in the web server or even the invocation of a program to be executed at the server site. This could be something such as database querying. So as Ceri pointed out HTTP is an application level protocol for allowing users to make requests of resource is to remote servers. HTTP was invented by Tim Berners-Lee on Robert Cailliau as part of the World Wide Web project, which began in 1990. It is strange to think now that as recently as that, the World Wide Web did not exist. And it's interesting to note that the project's aim was to build a distributed hyper media system with a similar user interface, so that people could access data stored remotely in servers on a network.

In this lecture, I'm going to explain a simple web server architecture. The basic elements of a web server are: hardware, an operating system, HTTP server, databases, and scripting language. Hardware, the operating system and HTTP server are the minimum requirements, but most servers include databases and a scripting language as well. Now let me explain each part. Hardware for a web server could be anything from simple, low-power devices like Raspberry Pi to large stacks of compact servers in an air conditioned room. For a small company, web server hardware could be a single computer. For a large company, each service may need one or more pieces of dedicated hardware. Some companies or even individuals pay a web hosting company. This is where you can rent hardware or a section of a large piece of hardware instead of having your own server. Web hosting companies offer a variety of hardware, including personal virtual servers, Cloud servers, and dedicated servers. These all have very different price ranges. Such companies usually provide both hardware and software. Apart from the hardware, all the rest of the web server architecture will create their web server software. This is the operating system, HTTP server, the database and the scripting language. The allocation of resources to your application and communication between hardware and software is the job of the operating system. For example, three categories of operating system for servers are Linux, Windows, and Mac-based OSS. So what about the HTTP server, database, and a scripting language? Any output from our database and a scripting language is turned into HTML output. Then the client's browser displays the HTML content as web pages. It is the job of an HTTP server to compile the results from these three elements in order to create the content that is going to be sent to the user. Configuration files in HTTP server control what options are available to the end user. HTTP servers allow us to share our webpages as creates and files with our end users. We need an HTTP server to view HTML and JavaScript files as a destination on a network. Otherwise, we can only view them on a local computer. You are going to be learning and experiencing the development of an HTTP server.

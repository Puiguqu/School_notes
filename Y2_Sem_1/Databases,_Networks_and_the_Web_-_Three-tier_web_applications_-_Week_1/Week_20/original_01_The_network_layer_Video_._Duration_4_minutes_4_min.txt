As we have seen in previous lectures, the network layer is between the transport and link layers in the TCP/IP model. IP addressing and routing happen in this layer, which is sometimes also called the Internet layer. Internet protocol or IP is the name of the protocol in the network layer. As you may have noticed, IP is the second part of the name of the model, TCP/IP. Internet protocol in the network layer handles sending data from a source to the destination, which happens via a series of devices called routers. Each computer on the Internet has a unique number or address called the IP address. In IP version four or IPv4, 32-bits of binary numbers are used to define an IP address. As you might know from other courses, 32-bits can store two to the power of 32 numbers, which is more than four billion. For the purpose of readability, an IP address is represented by four 8-bit sections or architects. The number in each section is between 0 and 255. This is an example IP address, 92.40.248.23. Do you know how to find the IP address of your device? There is a discussion on that later in this topic. Routing is a forwarding mechanism that carries packets from source to destination on the Internet. The packet switching mechanism explained previously in this topic is the foundation of the routing process. Routers receive packets and forward them to the next hop. This will continue until their packets arrive at their destination. This doesn't mean all routers need to understand the whole network. They pass packets to neighboring routers. Routers have a forwarding table or routing table and based on this, they choose the best outbound route for each network address. Routing algorithms run at routers to determine these parts. Routers also pass updated routes back and forth. If they don't know a network address, they ask their neighboring routers. When you do your reading this week on Page 5 of advanced Internet protocol services and applications, you can see an IP packet header format. If you remember what you learned in earlier lectures in this topic, you'll know that this header is added to the TCP header received from the upper layer, which is the transport layer. There are different parts in the IP header, including source and destination IP address, type of service, header checksum, Time to Live and more. The source and destination addresses are necessary to support IP addressing mechanism. Let me explain one more section of the header and you can study more in the readings. Infinite loops in packet routing could result in a crash network, which would cause huge problems to lots of people. So the header called Time to Live, or TTL, is a very simple yet efficient mechanism to prevent infinite loops when routing packets. TTL is initialized by a value. For example, let's say 25. Every time it is forwarded by a router, its value is subtracted by one. The next router checks the TTL value of the packet. Eventually, the value will reach 0. When a router sees this, it knows that this packet has been in the network for too long. In that case, the router can destroy it. When a router discards the packet, it sends a message back about it.

So far, you have created your web server and a simple web application. You have learned how to add new routes to your web application, and you have reorganized your web application considering the separation of concern design principle. Separating presentation, logic and several code in separate directories. Your HTML files, which are responsible for the presentation section of your code, are stored in the views folder. Your web server file, index.js, is in the home directory and your main.js file, which is responsible for main logic of your application, includes routing and is stored in the roots directory. Because the presentation part of your application includes more GUI elements and more interactions with users are included in it the HTML files get bigger and more complicated. Now think back to Topic 1. Do you remember the difference between dynamic and static web applications? As your web application moves away from a static app and towards a more dynamic one, you include more user interaction. These will result in HTML files that are presenting not only static content to users, but also presenting dynamic content, such as users input and database interactions and many more. That's how a templating engine comes onto the scene. A template engine can perform two main tasks. Firstly, imagine you would like to pass a variable such as a record of a data retrieved from a database to an HTML file and display it to the user. So passing a variable from back-end database tier to middleware, and then to a front-end presentation tier is something we would like to implement in our dynamic web application. A template engine will help us to do that. But how? By using a template engine, you will use static template files in your web application, including a variable names. The interesting thing about a template engine happens at run-time when it replaces variables in a template file with actual values and transform the templates into an HTML file. This is then sent to the client. This approach makes it easier to design an HTML page and provide a proper presentation to the end users. So a template engine can insert variables into the final output of the template. There is also one more benefit of template engines. They also enable you to run some programming logic at run-time before sending the final HTML to the browser for display. Imagine that you have a list of data retrieved from a database instead of a one single record of data. You can have a loop as a simple logical elements in your HTML file and the templating engine will repeat this loop at run-time based on values determined and assigned to variables. Always remember to keep the logic in HTML files to an absolute minimum. This is because of the separation of concerns principle. Logic is supposed to be separated from presentation. The main purpose of HTML file is presentation, and we keep the logic in the HTML file to an absolute minimum. Front-end or full-stack developers use defined tags to insert either variables or logic into HTML files. A template engine as a valuable tool will transform HTML files by replacing variables with values and running the logic at compile or run-time. I will introduce some of these defined tags later on. But since there is more than one template engine available, let me introduce a couple of them first. EJS, Pug and Mustache are some popular template engines for express and Node.js. You can choose from these options for your templating engine. But for this course we are going to use EJS. EJS stands for Embedded JavaScript. It is a simple template language for generating HTML markup with plain JavaScript. EJS template files contain, takes generally HTML in addition to a special EJS tags that led to embed the JavaScript expression into your template. The evaluation or returned value of JavaScript code will be added to the template at compile time. When you use EJS as a templating engine, you need to rename your HTML files as EJS files. The format of the HTML file is the same, but you have the option to add predefined tags for passing variables and incorporating logic. The two EJS tags, you need to learn about are firstly, This Tag. This Tag is used to output the return value of an expression into the document. For example, this will add two to the document. The other one looks like this. This Tag on the other hand, evaluates an expression but does not add the return value to the document. So this defines a name variable, but doesn't add the return value of the expression to that document. In this slide, you can see an example of an EJS file. As you can see, the value of title variable is replaced and displayed at run-time. There is also a loop as a simple logic in our template file that is repeated based on the values calculated at run-time. We'll be saving our EJS templates in the views directory for all our Express applications. Pug is another templating language for server-side JavaScript with Node.js. Previously it was referred to as Jade. It has a whitespace and indentation sensitive syntax like Python for writing HTML. Pug is high performing and feature-rich. It is very clean, meaning all repetitive HTML tags are removed. It is so clean and simple and is one of those things that you either love or hate. I usually ask my students and some of them love it and some do not like it at all and prefer to see HTML tags when they open a template file. I will give you references in your readings to see examples of Pug with corresponding HTML files to compare it with. It would be great to hear what you think of Pug in the discussion forum. One, passing variables to HTML files and then adding simple logic to them.

That is, one where data from the middleware has been inserted into the HTML response. Let's take a little time to understand how this code is working. Previously, we saw how HTML files can be rendered using EJS. This HTML file is static, in that its content is always the same. We can start to make things dynamic by replacing some hard-coded text with an EJS tag, and passing data to the EJS page. Here in main.js, we are defining a new object called title, and passing that object to the EJS file where it is rendered. In the EJS file, we can use an EJS tag that inserts the value of the title object when the page is rendered. EJS tags are always enclosed with these symbols. Inside the tag, we can write JavaScript code. When we see the equal sign at the start of the tag, EJS will insert the value of the JavaScript expression in the tag. In this case, the expression is simply the value of the title variable. So in our code, when the home page is requested, the browser tab title is dynamically set from the JavaScript object title. If the value of the JavaScript object changes, the title shown in the browser will change too. This gives the ability to drive the content of our web pages from data within the middleware of our web application. As we will come to see in future labs, this is a very powerful technique.

Each article is a concept, but each author is also a concept. An information about both concepts is stored within each list entry. These examples show the modification issues associated with a list of data. Redundancy and multiple themes in list create modification problems with three operations we might need to perform insertion, deletion and updates. When we insert a new article data redundancy occurs for all articles written by the same author. When we are deleting an article, we lose all the information about the author if that author hasn't written any other articles. When we are changing or updating an author for an article, more than one field has to be updated. So in summary we can see that tab delimited files or lists of data are not a good way to model the real world problem of data storage and retrieval because it is difficult to retrieve sets of related data quickly and there are issues when we need to modify the data in the list. Relational databases were designed to address these various problems. They are used in many applications and in all scenarios.

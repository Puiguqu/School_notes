As I said in the previous lecture, before relational databases were developed data had to be stored in linear files which were long lists of tab-delimited files. This meant it was very difficult to look up and retrieve sets of related data quickly. So the relational model was designed to make this process much simpler and quicker. So what is a relational database? It's actually quite a simple idea. In the relational database model, data is organized into tables which relate to each other. Relationships between tables are established or modeled by common values in related tables. According to the definition in modern database management, a relational database presents data as a collection of tables in which all data relationships are represented by common values in related tables. In a relational database, each type of object or entity is represented by a table. As you will recall, object here has a similar meaning as in object oriented programming or OOP and in many cases corresponds to real world objects or entities such as customers, products, students, universities, and so on. So a relational database will break up a list into several parts where each information theme or business concepts that we want to keep data on is stored in its own table which must have a unique name. For example, our newspaper articles list from the previous lecture might be divided into an article table and an author table. In relational databases, table rows are referred to as records and columns are called fields. Each column represents the different attributes of an entity and each row represents an instance of the entity. In the example shown here, you can see a table of the newspaper's customers. Each row or record represents data related to a particular customer and each column or field represents a different attribute of those customers, namely the customer ID number, first name, and last name. As you can see in this table, each row represents a unique instance of a customer. Relationships in relational databases are established by common fields in tables. In this example, when we're looking again at the author and article tables, their relationship between them is established by author name, which is a common field in both tables. As you can see, the modification issues associated with lists are solved. We do not have data redundancy, because there is a separate table for authors, so the information about each author is only recorded in that table and is not replicated in the article table. The insertion, update, and deletion problems we identified in our last lecture are also addressed. Do you think author name is a good link between two tables? There is a discussion activity after this lecture where you can share your answer to this question. Let's come back to what should be a familiar concept now, database management systems. As we know from our reading in modern database management, a database management system is a software system that is used to create, maintain, and provide controlled access to user databases. There are a number of DBMSs for relational databases including MySQL, PostgreSQL, Microsoft SQL Server and Oracle. SQL or structured query language, is a database processing language specifically designed to access and query data managed by relational database management systems. MySQL is a free and open source relational DBMS, which is based on SQL. Later in this topic, you will see lab instructions on how to use MySQL Shell.

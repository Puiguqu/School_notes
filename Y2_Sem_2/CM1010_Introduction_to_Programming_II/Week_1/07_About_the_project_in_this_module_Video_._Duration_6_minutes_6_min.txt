This redesigned module syllabus fundamentally shifts the learning experience compared to "Introduction to Programming Part 1," prioritizing independent code construction. The primary pedagogical goal is moving students past syntax support toward independently developing their own code and structures. The course emphasizes proficiency in thinking about problems and working towards algorithmic solutions. Crucially, students will be assessed through a large, project-based assignment designed to simulate advanced industry work. This project requires participants to master the full software development cycle (SDLC). The SDLC encompasses analyzing problems, designing iterative code structures, coding the solution, and finally, rigorously evaluating the outcomes. Assessment is multidimensional, evaluating not just the written code, but also the quality of brainstorming ideas and design documentation. Students must demonstrate an effective and iterative design process, showing critical thought applied to the problem. Furthermore, organization, encompassing both code structure and time management, is an essential assessment criterion. Critiquing one's own work during and after the development period is also mandatory.

Regarding academic grading, the pass line for an undergraduate qualification is set at 40% and must be achieved across both the midterm and final submissions. A score over 40% is considered a pass or satisfactory, potentially equating to a third class degree. Achieving over 50% is classified as good, corresponding to a 2:2 or second class lower division. The mark of 60% is categorized as very good, which is equivalent to a 2:1 or second class upper. First class honors are achieved with a score over 70%, indicating excellent performance. The academic scale protects the very top grades, meaning scores significantly beyond 70% (e.g., 80% or 90s) are highly unusual at this undergraduate stage. The module aims to reflect later degree assessments, with previous cohorts anticipating a class average of around 62% or 63%. Students should view a performance between 50% and 65% as representing good or very good work.

The initial modules focus on Object-Oriented Programming (OOP) principles. Week 1, for instance, introduces Object concepts and practices, covering topics like Visual Studio Code setup and a debugging challenge in a chocolate shop context. Subsequent weeks systematically build upon these core concepts through guided case studies. The curriculum introduces Case Study 1: the drawing app (Weeks 3-4), followed by Case Study 2: the music visualiser (Weeks 5-6). The next major focus is Case Study 3: data visualisation (Weeks 7-8). The core of the module involves extending these apps through Parts 1, 2, and 3 (Weeks 9-14). Technical topics later covered include advanced concepts like Callbacks (Weeks 15-16). The latter stages of development focus heavily on quality assurance, covering Topic 9: Testing for stability (Weeks 17-18), and Topic 10: Testing with users (Weeks 19-20). The module culminates in Week 21 with the final Project and report assignment.

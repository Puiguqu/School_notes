The project builds upon previous flying saucer examples, incorporating a newly added `cow` object into the starter sketch. Technical details and all stages of the build are accessible via the `T1.205 ZIP File`. Students must first analyze the provided `cow` constructor to understand its functionality. Within the `flying_saucer` constructor, specific modifications are required for local variable handling. First, "brightnesses" must be changed from a stored property to a local variable within the constructor. Furthermore, the `beam` method itself must also be converted into a local method. A new variable, assigned the value of this localized beam method, must be created and called `self`. The central concept involves creating a dedicated object called `CowManager` to handle cow interactions. This manager object must contain an array to hold multiple `cow` instances, which should be maintained as a local, private variable. The `CowManager` requires both a `draw()` method and an `update()` method implementation. Within the `CowManager`, a variable named `minCows` must be established to ensure a minimum number of cows is always present. The `update()` method must rigorously enforce this minimum cow count. Using a `for` loop is required to call the individual `walk()` method for every cow instance. The `draw()` method must iterate through the array to call the individual `draw()` method for all cows. The main sketch needs updating to utilize a new instance of the `CowManager` object. A critical improvement for the `CowManager.update()` method is preventing cows from exiting the screen boundaries.

For the second phase, utilizing the `cows_3` sketch, the tractor beam capture mechanism is implemented. The `levitateCows` function has been provided, along with initial necessary code. The existing code must be updated to manage multiple flying saucers. This requires the use of a global array, specifically named `flyingSaucers`. In the main sketch’s `draw()` method, the code must traverse this `flyingSaucers` array every frame. This traversal necessitates calling the `update()` method for every flying saucer within the array. This structured approach significantly scales the object-oriented system by allowing multiple independent saucers to operate simultaneously.

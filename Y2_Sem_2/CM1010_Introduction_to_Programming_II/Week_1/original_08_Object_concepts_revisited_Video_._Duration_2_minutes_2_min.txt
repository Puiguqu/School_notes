And as the things we created became more complicated, it became neater to group those variables together into objects. A general rule of thumb was that an object should be used to describe a singular entity within our programs. What were formerly simple variables became properties of an object. Now, we could change those properties to create variation. After a while, we wanted our objects to have behaviors and so we used methods, and we called these just like functions. But if we needed to refer to the object inside a method, we had to use a special keyword, this. We could think of the keyword, this, as meaning me, my or I. Finally, we wanted to create multiple versions of our objects. So we created constructor functions, and we use them with the new keyword. This allowed us to generate many objects on the fly.

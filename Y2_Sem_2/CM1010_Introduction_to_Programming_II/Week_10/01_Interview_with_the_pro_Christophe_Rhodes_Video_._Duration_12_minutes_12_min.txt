# Interview with the pro: Christophe Rhodes

Christophe Rhodes, a Senior Software Engineer (SWE) at Google, works on the Android Studio IDE, which is the primary programming environment for Android developers globally. An SWE's duties involve writing code and supporting developers building functional and usable Android products. Key programming languages required for the role include Java, an established industry standard, and Kotlin, which is highlighted as potentially the next standard language due to its compatibility with Java and modern features. Beyond coding, proficiency in debugging is critical, often requiring the identification and reproduction of user-reported problems to find their root cause. Furthermore, developers must utilize measurement tools, such as a profiler, to determine performance bottlenecks, distinguishing between minor tuning and fundamental architectural rewrites. The professional workflow relies heavily on specific infrastructure tools, including version control using GIT for managing all changes. Developers must also be proficient in building systems like Bazel and Gradle, and writing automated tests to prevent future functional errors. The learning process at Google is multifaceted, combining intensive induction, self-study materials, and leveraging the team as a collaborative resource. Mentorship typically involves reviewing changesets to guide the developer toward understanding the underlying architecture and programming language details. Work mechanics dictate that proposed changes are developed as "change lists," which are then sent for comprehensive code review. These reviews are generally standardized, allowing humans to concentrate on design quality and functional improvement rather than superficial stylistic rules. Rhodes notes that the time spent coding is relatively small compared to the time spent reviewing code and debugging. Debugging is described not as writing code, but as a process of reading code and analyzing the program's state to find unexpected behavior. Moreover, designing a new feature requires writing and reviewing formal design documents, often involving specialists like UI/UX experts. A central guiding principle in programming is that accuracy must always supersede speed, ensuring defect-free implementation for the end-user. To succeed, students must focus on developing flexibility, not just mastering a single language or tool. They should seek exposure to various object models and programming languages that express solutions differently from the one they are initially learning. The curriculum supports this depth through topics like "Object orientation in practice" in Weeks 1 and 2. Specific case studies are planned, including "drawing app" (Week 3/4), "music visualiser" (Week 5/6), and "data visualisation" (Week 7/8). Advanced technical concepts are covered in subsequent weeks, including extending applications (Weeks 9-14) and understanding "Callbacks" (Weeks 15/16). The schedule also dedicates time to critical engineering processes such as "Testing for stability" (Weeks 17/18) and user testing (Weeks 19/20). Final work requires completing the project and report assignment, solidifying knowledge gained across diverse technical domains.

The objective of this extension is to add tweakable controls to the audio visualizer using a Graphical User Interface (GUI). For this, the speaker utilizes the `p5.gui` add-on available on the p5js website. The initial steps involve retrieving the necessary library files by clicking "Clone or download" on GitHub and selecting the "ZIP" option. Careful review of the documentation reveals that two specific files must be included: `p5.gui.js` and `quicksettings.js`. When coding, these relevant files must be copied into the project's libraries folder. Next, the script tags for both `p5.gui.js` and `quicksettings.js` must be added to the `index.html` file. A critical technical detail is that `quicksettings.js` must be loaded *before* `p5.gui.js` for the code to function correctly. The GUI is initially created within the `setup()` function using the `createGui()` method. To expose controllable variables, the `addGlobals()` method is used, taking the variable name as an argument. For instance, setting a global variable like `noiseStep` allows real-time adjustment of the graphical parameter. Because the default slider range may lack sufficient granularity, the `slideArrange()` function is employed for fine-tuning. This function requires defining the minimum value, the maximum value, and the desired step size for the control. The speaker demonstrates using `slideArrange()` to set a minimum of 0.0001, a maximum of 1, and a step of 0.001 for better control. Additional adjustable variables, such as `rotateThresh` and `progThresh`, are implemented using this same advanced `slideArrange()` command structure. Another significant finding is adjusting the visualization parameters to use different energy inputs, specifically passing in the `treble` energy rather than the default `bass` energy. When a global variable is added, it is crucial to use it within the sketch's code to observe any visual effect. For example, merely creating `rotateThresh` does not change the visuals until the variable is explicitly used in a calculation. The visualization can be developed further by adding secondary energy analysis, such as calculating a `seedThresh`. This technique involves monitoring a second energy value that triggers a randomizing function, like the `noiseSeed()` function. The speaker concludes by successfully integrating all these global variables and refined controls, resulting in a fully adjustable and complex audio visualizer application.

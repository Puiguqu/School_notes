# Asynchronous function calls

1. The core technical concept discussed is the distinction between synchronous and asynchronous function calls in JavaScript.
2. In JavaScript, function calls are fundamentally divided into these two categories: synchronous and asynchronous.
3. Synchronous function calls dictate that the program must execute each line of code sequentially, one after the other.
4. For instance, calling a function like `ellipse` within a draw loop causes the draw code to pause until the internal execution of the `ellipse` function is complete.
5. When the function finishes, the draw code automatically resumes execution on the next line.
6. Conversely, asynchronous functions behave differently from synchronous ones, allowing multiple tasks to proceed concurrently.
7. An example of an asynchronous function is the `loadSound` function, particularly when called within the `preload` function.
8. When `loadSound` is called, the `preload` function does not halt; instead, it continues executing simultaneously with the sound loading process.
9. The transcript warns that while asynchronous functions are generally harmless, they can cause significant issues if developers assume immediate completion.
10. A concrete example of a problem is demonstrated when attempting to retrieve a loaded sound's duration immediately after calling `loadSound`.
11. If the code attempts to access the sample duration before the asynchronous loading completes, it will incorrectly report a duration of zero.
12. This happens because the main code thread continues running, making the duration property inaccessible or unset until the sound loading is finished.
13. The speaker notes that although the displayed duration is zero, the underlying audio buffer and its length may exist when checked in the console.
14. Therefore, accessing properties (like `sample.duration`) immediately after an async call creates a race condition.
15. A simple workaround shown is setting a flag or variable (e.g., `duration = sample.duration`) only when a specific user action, like pressing a key, occurs.
16. This delay ensures the variable is assigned the correct sample duration value at the point of need.
17. The speaker emphasizes that this simple fix is temporary and that a much better, more robust solution will be covered in subsequent videos.
18. The primary question addressed is why asynchronous functions are necessary if the program could simply wait for every job to finish.
19. While waiting would simplify programming, asynchronous processing is crucial for efficient computation.
20. The efficiency gain is realized because the computer's processing power can be utilized for other useful activities while certain tasks (like file loading) are ongoing.
21. This ability to multitask prevents the application from freezing or appearing unresponsive ("spinning wheels").
22. Ultimately, asynchronous programming leads to less frustrating and more responsive software experiences for the user.

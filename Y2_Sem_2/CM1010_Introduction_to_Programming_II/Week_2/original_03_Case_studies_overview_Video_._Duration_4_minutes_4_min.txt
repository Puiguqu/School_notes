In this topic, we have already discussed the need for object orientation and dividing your code into manageable chunks. This should help you to organize and navigate through the codebase. Also, through this process, you should be better able to interpret the code written by others. You'll be putting together a big project as part of this module and this will make up the various elements of your assessment. The project will be based on one of three case studies that we are going to present over the next three topics. We will be exploring three bare-bones applications written in JavaScript using p5.js. Each of these applications has made use of these techniques and many other language features we taught you in introduction to programming Part 1. They have been constructed so that they can be extended with new features, tools, and functionality that you'll be designing, implementing, and testing. Over the next few topics, we will get you started on each of these applications explaining its composition and structure. Each one uses constructor functions and object orientation as well as a new p5.js external library or object in its implementation. There'll be an exercise for you to complete with each case study that will help you to make sure you have understood the application's code. This will be validated by your classmates with a peer-review. It will then be up to you to create compelling creative extensions and new tools for the application. You'll be assessed not just on the technical implementation and the complexity of your code, but also on the planning, organization, and evaluative skills you employed during the applications development. This is not just about coding but being an effective coder. The first of these applications which we will introduce you to in the next topic is a simple drawing application. Similar in functionality to MS Paint and Apple's Mac Paint, the drawing application makes use of the p5.js DOM library to access elements of the web page outside of the Canvas to handle more complex controls and layout. The apps toolbox has a few basic tools for drawing wiggly and straight lines and an airbrush too. Extensions to this application may create a richer set of tools with more customization options. In Topic 3, you will complete the music visualization app. This app makes use of the p5.js sound library that you may remember from the game project in introduction to programming Part 1. As well as handling music and sound playback, this library can be used to convert the properties of the output sound waves into values which can be used to create visual patterns, representations, and works of art. Finally, in Topic 4, you'll be introduced to a data visualization app. It's an increasingly important part of journalism and public understanding to be able to communicate complex numerical data using graphs and other meaningful representations. We'll be using the p5.js table object to handle and manipulate tabulated data loaded from a comma separated value file. After you have completed the case studies, you'll have to pick one to be the basis of your project.

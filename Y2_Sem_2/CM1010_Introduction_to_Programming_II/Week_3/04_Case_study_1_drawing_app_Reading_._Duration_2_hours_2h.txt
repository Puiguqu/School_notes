The drawing application case study requires operating the application within a development environment like Visual Studio Code or Brackets. Initial setup involves dragging the ZIP file folder into the editor and clicking the live preview button to launch the app in a browser. Users can initially interact with four tools: freehand, line draw, spray can, and mirror draw. Before modifications, it is crucial to thoroughly read and understand the various constituent project files. The structure utilizes the Document Object Model (DOM) extensively, making `index.html` complex and built outside the canvas area. The `style.css` file handles the application's overall layout and visual styling via the DOM. Core logic resides in `sketch.js`, which handles global variables, the `setup()`, and `draw()` functions using the p5.js library. Container files like `colourPalette.js` and `toolbox.js` manage the application's state, such as identifying the currently `selectedTool`. The `helperFunctions.js` file is designated for utility functions and currently holds event handlers for 'clear' and 'save image'. Individual tool files, such as `lineToTool.js`, must adhere to a specific structure. All tools require three properties: an `icon` (a URI for the tool icon), a `name` (a string identifier), and a `draw` function.

Task 1 focuses on documenting `lineToTool.js` to ensure clarity for other programmers. Within this file, one must investigate the functionality of `loadPixel` and `updatePixel`, referencing the p5 documentation. A critical diagnostic step is attempting to comment out these functions to observe their impact on the program's execution. Task 2 requires refactoring the spray can tool's architecture. The existing object literal in `sketch.js` must be extracted and rewritten as a dedicated constructor function. This new `SprayCanTool` constructor function must reside in a separate file, `sprayCanTool.js`. Implementation requires defining properties and methods using the `this` keyword (e.g., `this.points = 13` and `this.draw = function()`). Subsequently, `sprayCanTool.js` must be linked in `index.html`, and `sketch.js` must be updated to initialize the tool using `toolbox.addTool(new SprayCanTool())`.

Task 3 involves completing the core utility functions within `helperFunctions.js`. The 'clear' button handler (line 9) requires replacing `???` with the p5.js `background()` function call to reset the color to white. For the 'save image' button handler (line 19), the missing function call must save the canvas to the hard disk, as guided by the p5.js reference guide. Further enhancements encourage analysis of the complex mirror tool's mechanics. Students are also encouraged to implement original tools, such as a rectangle or ellipse tool. Upon completing the case study, the final sketch should be uploaded for peer review, ensuring full understanding of all technical tasks.

The core functionality of the drawing application relies on specific programming commands within the p5.js framework, namely `loadPixels` and `updatePixels`. The `loadPixels` function is crucial because it captures and saves a complete, current copy of the drawing's pixels into the computer's memory. This saved copy effectively establishes a stable, background state for the drawing application. Conversely, the `updatePixels` function utilizes this saved data, restoring the background image to its initial saved state during every drawing cycle. This mechanism functions essentially as a programmable "undo" feature for the application. The technical implementation of the line tool exemplifies this state management process. This tool utilizes three primary variables: `drawing`, `startMouseX`, and `startMouseY`. The variables `startMouseX` and `startMouseY` are initialized to -1, serving as a sentinel value. When the user first clicks down, this sentinel value triggers a critical setup phase. During this phase, the variables `startMouseX` and `startMouseY` are reset to the current `mouseX` and `mouseY` coordinates, thus recording the starting point of the line. Following this setup, the program sets the `drawing` variable to `true` and immediately executes `loadPixels`. This action permanently stores the current image state before any line drawing occurs. When the line is being actively drawn, the program repeatedly calls `updatePixels` every single frame. This continuous call ensures that the background is reloaded with the initial saved image state. Simultaneously, the line segment is then drawn precisely on top of this reloaded base image. This cyclical process maintains the illusion of drawing on a stable background. Once the user releases the mouse button, the drawing process concludes. The code then executes several cleanup actions to reset the state. Specifically, it sets `drawing` back to `false` and resets `startMouseX` and `startMouseY` back to -1. These resets ensure the line tool cannot re-engage until the user initiates a new click event.

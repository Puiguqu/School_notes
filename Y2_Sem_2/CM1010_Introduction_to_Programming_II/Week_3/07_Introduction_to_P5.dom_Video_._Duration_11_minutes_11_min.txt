1. The p5.js DOM library was updated to integrate previously separate features, eliminating the need to download or include a separate DOM library file.
2. The DOM (Document Object Model) adds specific functionality to p5.js by providing access to parts of the HTML web page outside of the p5 canvas.
3. DOM represents the structure of a web page, encompassing elements like blocks of text, buttons, and controls.
4. Web pages are built as a tree structure of different elements, which is the core concept of the DOM.
5. An HTML element starts with an opening tag, placed in angle brackets, and typically ends with a closing tag containing the same tag name with a forward slash.
6. The primary structural tags include `div`, which acts as a divider to portion the page into sections (e.g., navbar, sidebar), and `p` for paragraphs.
7. Image elements use the `<img>` tag, which is unique as it does not require an end tag because it lacks inner content.
8. To manipulate HTML code, developers use attributes such as Classes and IDs, which assign specific names to elements.
9. An ID attribute specifies a unique name for an element, ensuring only one instance of that ID exists on the page.
10. In contrast, a Class attribute allows for multiple elements to share the same designation, enabling bulk styling or manipulation.
11. When accessing these elements in JavaScript, a period (`.`) must precede the name for a class, and a hash sign (`#`) must precede the name for an ID.
12. Before using the library, the p5.dom.min.js file must be imported into the index.html file, placed in the specific order after p5.min.js but before sketch.js.
13. p5.js DOM facilitates two fundamental operations: creating new elements and accessing pre-existing ones.
14. Functions like `createButton(text)` and `createDiv(html)` are used to programmatically build elements, allowing for content generation.
15. The programmatically created elements, such as the canvas, div, and button, are added to the web page structure in the index.html.
16. To retrieve an element, the DOM library employs the `select()` function, which accepts an ID, class name, or tag name as its single argument.
17. If searching by ID, `select('#myId')` returns the specific element object, or `null` if the element is misspelled or missing.
18. When using a class or tag name with `select()`, only the first instance found on the page is returned.
19. To target all elements sharing a class or a tag, the more comprehensive `selectAll()` function must be used, which returns an array of elements.
20. Once an element is selected, programs can utilize its properties, such as checking the `value` of a control like a button or text area.
21. Furthermore, developers can alter the styling of an element or change its underlying HTML content via the selected object.
22. The entire web page structure is a hierarchical representation known as the Document Object Model (DOM).
23. The `select` function allows interaction with either elements created by p5.js DOM or elements initially written into the index.html.
24. Understanding HTML basics is crucial, as professional web development requires building websites manually in a code editor.
25. Mastering the p5.js DOM enables the creation of sophisticated drawing applications and interactive web content outside the primary canvas.

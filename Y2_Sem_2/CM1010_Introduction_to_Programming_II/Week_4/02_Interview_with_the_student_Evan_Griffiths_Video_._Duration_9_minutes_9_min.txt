# Interview with the student: Evan Griffiths

This presentation details extensions made to a template drawing application using object-oriented principles in practice. The student, Evan Griffiths, built upon the core template tools, including the pencil, line, spray, and mirror tools. A major functional improvement was the implementation of an HTML color picker, which vastly expanded color options beyond the original 12 choices. This color picker allows for fine control over drawing, affecting nearly all existing tools. Specific controls added include line weight, which manages the thickness of drawn lines, and line alpha, which dictates transparency for creating shades. Furthermore, the app was enhanced with an eraser tool and a shape tool utilizing vector graphics for scalable shapes (e.g., squares, stars). A robust text tool was added, allowing users to type and change text color on the fly, integrating with all existing drawing options. The most complex feature is the 3D model tool, which allows importing 3D models, specifically in the OBJ file format. Users can reposition, scale, rotate, and stamp these imported models onto the canvas. The model functionality includes the ability to recolor models using a color wheel and to import additional textures. Technically, the expanded color selection is managed by an HTML `<input type="color">` element. The associated hex code is parsed and then passed through a function that splits it into Red, Green, and Blue channels. The code then performs conditional conversion to a format understandable by the framework (originally using jQuery, but noting the current use of P5.dom). Model file structure is critical, as an OBJ file is relatively human-readable, containing vertices (V) defining points and faces. These vertices use X, Y, and Z coordinates to define the model geometry for rendering. To facilitate 3D rendering, the developer utilized the WebGL renderer in P5, changing the default 2D canvas context. This involved positioning the drawing area as a three-dimensional rectangle positioned near the camera. This clever setup ensures that drawing texture applied when clicking appears to function exactly like the original 2D drawing context. P5 handles the core mechanism of importing the 3D objects, allowing the developer to load the OBJ file into a variable for manipulation. Implementing a proper user experience required adding a visual indicator, such as a red highlight box, around selected 3D objects for transformation control. The developer noted that this entire process of integrating 3D objects and WebGL contexts is significantly more complex than standard 2D canvas operations. For future iterations, updating to the latest library version is recommended, as P5.js is under active development with frequent functional updates. Overall, the project successfully extended basic drawing functionality with advanced 3D manipulation capabilities and detailed technical integration.

# Interview with a student: Taylor Millin Read

The presentation focuses on a music visualization application built for a programming project, showcasing technical application of audio processing concepts. The system incorporates several visualizations, including "Spin," which reacts to the waveform and spectrum analysis derived from the Fast Fourier Transform (FFT) algorithm. Another feature is a "store waveform" display, and a third, more advanced application utilizes a pixel array to dynamically change colors as music plays. Taylor's favorite visualization is the video app, which leverages the pixel array for color manipulation. This process involves writing a function that iterates through the four elements comprising the red, green, blue, and alpha components of every individual pixel. Technical details for the video app require calling the `loadPixels()` function and subsequently updating the pixels using nested `for` loops before calling `updatePixels()`. The original inspiration for this video app was integrating a camera video feed, making it react fluidly to music changes. Regarding development limitations, the speaker noted the desire to explore color systems beyond standard RGB, recognizing other available color formats in p5.js. The "Spin" visualization demonstrates a simple yet effective method: translating everything to the screen's center and using rotational angles (`rotateAngle`) rather than complex trigonometric formulas. Furthermore, the overall control interface was built using fundamental principles: drawing triangles in the `draw` function and checking if a click event falls within the boundaries of these defined shapes. The educational curriculum structure progresses systematically, starting with topics like Object Orientation in Practice (Weeks 1-2). Early case studies include a drawing application and the deep dive into the music visualizer (Week 5). The module structure details preparing for the music visualizer using p5.js sound concepts, including amplitude and frequency analysis. Assignments cover reading and practice, ranging from understanding sound composition to analyzing the music visualization app "under the hood." Subsequent weeks broaden the scope, introducing data visualization in Week 7 and extending the application in Weeks 9 through 14. The curriculum then tackles advanced topics such as callbacks (Week 15), stability testing (Week 17), and user testing (Week 19). The final major deliverable is the Project and Report assignment, signaling the completion of the extensive module sequence.

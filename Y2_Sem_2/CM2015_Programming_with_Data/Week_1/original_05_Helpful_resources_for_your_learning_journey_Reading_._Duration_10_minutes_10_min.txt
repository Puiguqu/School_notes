# Helpful resources for your learning journey
Reading•
. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-cm2015-programming-with-data/supplement/gvu01/helpful-resources-for-your-learning-journey

As you progress through this course, you’ll encounter resources designed to help enhance your learning experience and equip you with practical skills. To help you succeed, we’ve compiled a guide of key tools and strategies that will support your understanding of the material and your growth as a programmer. Revisit video tutorials: The course videos provide step-by-step explanations of key concepts and coding techniques. Don't hesitate to pause and rewind if you need extra time to digest the material! Using official documentation pages: Python documentation, as well as documentation for key libraries like pandas and NumPy, are incredibly helpful resources. They usually include worked-out examples that show how to write and use a specific piece of code. When consulting the documentation, look for a concrete example, try running it in your Jupyter Notebook, and experiment with modifying it to see how it works. This hands-on approach will help you better understand how the code is structured and applied, reinforcing the concepts from the course. Reading materials: The reading materials assigned in this moduleare carefully selected to complement the video lectures. They contain code snippets that illustrate the same concepts you encounter in the videos, but on a different problem. By connecting the readings with the course material, you'll be able to explore how to approach the same problems in various ways, which can deepen your understanding. Check the forum: If you’re stuck or need further clarification, check the forum. Chances are that someone has already asked it. While posting code questions, focus on describing the error message or the specific problem rather than posting the code. Provide context by including what you’ve tried so far, any error messages, and a summary of the issue. This will help others understand your problem and offer more targeted advice or solutions. Peer feedback: Engage in meaningful feedback exchanges. A great way to provide feedback is to start with what was great about their work, then offer suggestions on what could be improved. For example, you could say, ‘I really liked how you used pandas for data cleaning; it was efficient and well-organised. One thing to consider improving is adding more comments to explain your logic in the code – this will help others follow along more easily.’ Happy coding!

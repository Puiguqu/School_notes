Web scraping is defined as using a program to download and process content originating from the web. In this topic, learners will focus on two main methods of acquiring web data: scraping raw HTML content from websites and retrieving data from public Application Programming Interfaces (APIs). Understanding the basics of HTML is crucial, as an HTML file is fundamentally a plain text file with the `.html` file extension. Within these files, content is structured by tags, which are words enclosed in angle brackets that instruct the browser on page formatting. All valid HTML files must begin with the `<html>` tag and close with a corresponding ending tag. A starting tag and a closing tag together enclose text, forming an element. The content located between these tags is referred to as the inner HTML, or text. A key example of an HTML tag is `<a>`, which is used to enclose text that functions as a hyperlink. The specific destination URL for this link is defined by the required `href` attribute. To inspect the underlying structure of any webpage, users can right-click and select "view source" to see the actual HTML text received by the browser. Beyond HTML, many websites offer public APIs, providing data feeds in formats like JSON. For accessing these APIs, the `requests` package is the recommended Python method. This same `requests` package can also be utilized for fetching HTML data from various websites. The overall data workflow involves sending requests, parsing the resulting data (whether JSON or HTML), and subsequently loading it onto the Pandas library. After parsing, the structured data is typically stored as a CSV file. Weeks 9 and 10 dive into advanced text and language processing, respectively. The syllabus concludes with specialized data handling skills, including data cleaning (Week 15), data transformations (Week 16), and advanced data analysis and visualization (Week 17).

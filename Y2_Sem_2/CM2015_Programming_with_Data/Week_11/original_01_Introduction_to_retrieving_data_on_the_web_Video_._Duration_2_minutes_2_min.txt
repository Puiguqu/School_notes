For example, Google runs many web scraping programs to index web pages for its search engine. In this topic, you will learn to use several libraries that make it easy to get data from the web. We will look at two main sources of data from the web. Scraping HTML data from websites and getting data from public APIs. In case it's been a while since you've looked into any HTML, here is a quick overview of the basics. A HTML file is a plain text file with the.html file extension. Text in these files is surrounded by tags, which are words enclosed in angle brackets. The tags tell the browser how to format the web page. All HTML files start with the HTML tag and a corresponding closing tag at the end. A starting tag and a closing tag can enclose text to form an element. The text or inner HTML is the content between the starting and the closing tags. There are many tags in HTML, let's look at one of it. The tag a encloses text that should be a link. The URL that the text links to is determined by the href attribute. For any web page, to view the HTML of the page, right click on your web browser and select view source or view paid source to see the HTML content. This is the text your browser actually receives and renders. Another source of data we will look at in this topic are APIs. Many websites have public APIs providing data feeds via JSON or some other equivalent format. There are a number of ways to access these APIs using Python. One method that I recommend is the requests package, which can also be used for getting HTML data from websites. In the following videos, let's learn the libraries for sending requests to get HTML, as well as JSON data, parse it and work with these in a Jupyter Notebook.

Exploratory Data Analysis (EDA) is presented as a comprehensive, multi-part activity spanning weeks 6 through 10. Each topic covered during these weeks corresponds to a specific, sequential component of the overall EDA process. This entire activity is structured as a single Jupyter Notebook for seamless execution and consolidation of learnings. The workflow begins with data acquisition, specifically scraping data from HTML formats. Subsequent steps involve joining this data with external databases. Following data joining, the process mandates meticulous data cleaning techniques. Crucial cleaning steps include handling missing values and identifying outliers. The analysis then progresses to visualizing data and exploring insights through various visualization methods. Finally, the comprehensive process culminates in combining all learned techniques. The ultimate goals are to make robust inferences and answer specific questions using the processed data. The first component focuses entirely on retrieving raw data from an HTML source. This crucial task requires converting unstructured HTML data into the structured format of a Pandas DataFrame. To begin, several necessary Python libraries must first be executed and made available to the user. The core tools employed for this phase are BeautifulSoup and the Pandas library. BeautifulSoup is used to parse the HTML structure into a traversable `soup` object. Specifically, developers must utilize the `find` or the `findAll` method within the soup object. These methods allow for the targeted extraction of relevant data points from the source code. The data intended for this EDA activity is housed internally in a variable named `EDA_data`. This variable simulates data that would normally be scraped from a live HTML webpage. Users must execute the cell that stores the data into the `EDA_data` variable first. Once this input data is secured, the next step involves loading the HTML data directly into Pandas. The primary objective is to transform the HTML data representation into a functional Pandas DataFrame. The expected output is a clean, structured Pandas DataFrame artifact. This final DataFrame must possess six distinct and required columns. These columns are specifically named: `name`, `price`, `in stock`, `rating`, `review`, and `launch date`. Successfully completing this task validates the initial, critical stage of the entire data analysis pipeline.

# Manually creating a database table

SQL (Structured Query Language) is a programming language used for creating, storing, and retrieving data from relational databases. Relational databases organize data into distinct rows and columns. The process begins by using the `sqlite3` module in Python for database operations. First, one must establish a connection to the desired database, which can be local or remote. To connect to a local database, the `sqlite3.connect(database_name)` command is utilized, returning a connector object, which is stored in a variable (e.g., `CONN`). This connection step is foundational for linking to the database. However, a cursor object is required to actually execute, modify, or create tables within the database. The cursor acts as a handle or interface for all subsequent database operations. To obtain this necessary cursor, the command `CONN.cursor()` is executed. All commands that create, edit, or modify the database are referred to as queries in SQL. These queries require a specific syntax and are executed using the `cursor.execute()` method. To run a query, the SQL statement must be passed within the `cursor.execute()` function.

Initially, attempting a query like `SELECT * FROM items` fails because the database table does not yet exist. Therefore, creating the table structure must precede data retrieval. To build the database, one uses the `cursor.execute()` command with a `CREATE TABLE IF NOT EXISTS` structure. This command defines the database schema, which consists of columns and their corresponding data types. We established a `product` table within the `items` database. The first column, `product_id`, must be defined as an `INTEGER` and set as the `PRIMARY KEY` for unique identification. The `name` column is set as `TEXT` and must be constrained as `NOT NULL`. The `price` column is defined as a `REAL` or `FLOAT` type and is also `NOT NULL`. For stock tracking, the column type is `BOOLEAN`, and it is also marked `NOT NULL`. Finally, the `launch_date` column is established as `DATE` and constrained as `NOT NULL`.

After executing the creation query, all modifications to the database must be finalized using `connection.commit()`. To read the data, the `SELECT * FROM items` query is executed again. The results are stored in a variable (e.g., `data`) using `cursor.execute()`. To retrieve all the stored values, the `cursor.fetchall()` method is called. While `cursor.fetchall()` may return an empty array if the database is unpopulated, `cursor.description` confirms the existence and structure of the created columns. The next step involves learning full CRUD operations: creating, reading, updating, and deleting values in the database.

Data merging is reviewed through worked examples involving data from CSV and SQL databases. The initial focus is on a "simple merge" where data indices are perfectly aligned. For the simple merge example, the CSV file contains fields like `id`, `name`, and `age`. Concurrently, the SQL database holds data with fields including `id`, `department`, and `salary`. Crucially, the common field linking these two datasets is the primary key, 'id'. To combine this data, one must merge all columns that align based on the 'id' key. The specific code to perform this action is `merged_df = pd.merge(csv_df, sql_df, on='id')`. This operation successfully creates a new DataFrame incorporating all original columns, matched by their respective 'id' values. The resultant output clearly illustrates this alignment, including rows for Alice, Bob, Charlie, David, and Eva. In specific, complex scenarios, the process may require advanced join types, such as left, right, or outer joins. A second, more complex scenario involves "merging with filtering," which is necessary when the datasets do not match perfectly. In this advanced example, the primary keys are identified as `EmployeeID` and `ProjectID`. However, the immediate IDs present in the source data do not facilitate a direct merge. The actual common keys that enable the merge are the `EmployeeID` and the `LeadID` fields. The first dataset (CSV) is filtered, retaining only records corresponding to common IDs. The filtered CSV data includes entries for Bob, Charlie, and Eve. Simultaneously, the second dataset (SQL) is also filtered to select matching IDs. The filtered SQL data includes project details for Project A, Project B, and Project C. To execute the filtration process, users should refer to pandas operations for DataFrame slicing, and SQL query operations for selecting data. Once the data is filtered down to these common values, a merge can proceed using the established alignment between `EmployeeID` and `LeadID`. The resultant dataset successfully combines the data, showing records for Bob, Charlie, and Eve. For instance, Bob's row links him to Project A (ProjectID 101). This process demonstrates the critical step of pre-filtering both sources to ensure they match on a common key. The final merged table structure includes `EmployeeID`, `Name`, `Department`, `Salary`, `ProjectID`, `ProjectName`, and `Budget`. This comprehensive procedure ensures accurate integration of data from two sources with misaligned initial primary keys.

The following examples show how to avoid common mistakes and ensure consistency in data cleaning. Data type handling: Ensure that columns are converted to the right data types. Leaving a date column or a float or integer column as a string can cause operation like filtering by date or sorting to fail. Example: You have a column called date with values like "2023-01-01", but it’s stored as a string (object type). If you try to filter for a specific month, it won’t work as expected. What to do instead: Convert the column using pd.to_datetime() and print df.dtype to verify. 1 2 df['date'] = pd.to_datetime(df['date']) print(df.dtypes) # Verify the type of the column. Now, you can easily filter by month: 1 df[df['date'].dt.month == 1] Copy before removing values: Modifying a DataFrame without creating a backup could accidentally change your original data.To avoid this, save your cleaned DataFrame into a new variable or create a copy of the DataFrame before removing or replacing rows with missing values. Use the following to save the cleaned DataFrame to a new variable: 1. Use df.copy: 1 2 cleaned_df = df.copy() cleaned_df.dropna() 2. (OR) Store in a new variable: 1 cleaned_df = df.dropna() Verifying data after cleaning operations: Once you fill missing values with fillna(0) method, ensure that all the NaNs were replaced. To verify the changes, use the isnull and sum function to check if there are still missing values. Example: 1 2 3 df[price'].fillna(0, inplace=True) # Filling with 0 print(df['price'].isnull().sum()) # Check if there are still missing values Saving cleaned data: Finally, always save your cleaned DataFrame to a csv file to ensure that you do not lose the changes when you your script. Example: Clean and save data: 1 2 cleaned_df = df.dropna() cleaned_df.to_csv('cleaned_data.csv', index=False) This way, you can load it later: 1 df = pd.read_csv('cleaned_data.csv') Review of programming activity The following examples show how to avoid common mistakes and ensure consistency in data cleaning.

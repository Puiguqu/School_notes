The process begins by installing a local version of Jupyter Notebook on a Windows machine using Miniconda. Specifically, the installer utilized must be for 64-bit Windows. After installation, it is crucial to double-click the executable and, if prompted, ensure the option to add Miniconda to the system path is selected. Next, the user must open the Anaconda Prompt or PowerShell terminal for environment setup, as this method is demonstrated over using the Anaconda Navigator. Before proceeding, it is recommended to verify the installed Python version by typing `python` and confirming the interpreter works. To step out of the Python interpreter environment, the user must execute `Control D` or `exit()`. The primary step for managing project dependencies is creating a dedicated conda environment. This is achieved using the command `conda create CM2015`, specifying the desired Python version, such as 3.12. Once the environment is created, the user must activate it using `conda activate CM2015`, which changes the terminal's base indicator.

Once the environment is active, packages required for data programming must be installed using `pip`, the Python package manager. Key packages include `numpy`, `pandas`, and `matplotlib`. After installing these necessary libraries, the primary Jupyter functionality is added via `pip install jupyter`. With Jupyter successfully installed, the notebook can be initiated by typing the `jupyter Notebook` command. It is suggested that the user utilize Chrome for optimal performance when running Jupyter Notebook. When creating a new Jupyter Notebook, the environment list should automatically appear, indicating available kernels.

The most critical step is linking the new environment to the Jupyter kernel system. This is done by running `IPython kernel install CM2015`, making the specialized environment available within the Jupyter kernel selection. After this link is established, the user must reactivate the environment via `conda activate CM2015` and relaunch Jupyter Notebook. Upon starting the new notebook file, the CM2015 environment will now appear in the list of available kernels. This dedicated CM2015 environment serves as a specialized workspace for all future course projects. Users can customize this environment further by continually utilizing the `pip install` command to add more packages as needed. By following these steps, the user successfully sets up a complete, isolated, and functional environment for data programming.

The curriculum covers Python programming, starting with fundamental concepts and progressing through advanced data processing topics. Week 2 focuses heavily on "Packages and virtual environments," which is critical for proper development. The practical demonstration highlights the necessity of using a dedicated Python environment, which is defined as a context containing an interpreter and installed packages. To initiate this setup, one must use the `conda create` command, followed by the desired environment name (e.g., `cm2015`). Optionally, users can specify a Python version, such as `3.12`, when creating the environment. After confirming package installation, the environment can be entered using the `conda activate` command, which confirms success when the environment name appears before the interpreter prompt.

Once the environment is active, the process of installing relevant libraries requires the use of `pip install`, as pip is the default package manager in Python. Running `pip install` followed by a package name (e.g., `numpy` or `pandas`) installs the package specifically within the currently activated environment. A key technical detail is that any package installed is strictly limited to that specific environment and will not be available in other, separate environments. To verify the installed components, the `pip list` command must be used, which accurately reports all installed packages and their versions, such as `matplotlib version 3.1.0`.

These locally installed packages are designed for use both in executed programs and in Jupyter notebooks utilizing the same environment for compiling code.

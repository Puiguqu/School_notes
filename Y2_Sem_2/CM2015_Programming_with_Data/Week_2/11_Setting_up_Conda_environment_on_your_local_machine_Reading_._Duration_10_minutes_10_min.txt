# Setting up Conda environment on your local machine

To manage multiple projects requiring disparate packages and libraries, clear demarcations are essential. This necessary isolation is achieved in Python through the use of "Virtual environments." The tool specified for this process is Conda, a package and environment management system. Conda allows users to create, list, remove, and update specialized environments. These environments can house different versions of Python and various required packages. Moving between these segmented environments is technically referred to as activating the environment. All environment creation and editing must be performed using the local machine's Terminal. We begin the process by verifying the initial Conda installation status. To verify Conda's presence, users must open a terminal, such as Command Prompt or the Anaconda Prompt. The specific command to execute for this verification is `conda --version`. A successful execution confirms the installation, displaying a visible version number. To establish a new environment, the user must employ the creation command. For example, to create an environment named "dp4ai" with Python 3.12, the command is `conda create --name dp4ai python=3.12`. Users should remember to replace `dp4ai` with their desired environment identifier name. Once the environment creation is complete, the next crucial step is activation. Activation is necessary to begin using the isolated environment. On the Windows platform, the activation command provided is `conda activate dp4ai`. Alternatively, on macOS or Linux operating systems, the activation command remains `conda activate dp4ai`. After successful activation, required packages can be installed directly within the new environment. Packages such as numpy, pandas, and nltk can be installed using the command: `conda install numpy pandas nltk`. To view a complete list of all available Conda environments, the user must run the command: `conda env list`. To exit or leave the current environment's scope, the deactivation command must be executed. The command to successfully exit the environment is `conda deactivate`. If an environment is no longer required for any projects, it can be safely deleted. To remove an environment, the necessary command structure is `conda env remove --name dp4ai`. This removal step is noted as optional, depending on project needs. The overall process ensures robust, isolated, and manageable project environments.

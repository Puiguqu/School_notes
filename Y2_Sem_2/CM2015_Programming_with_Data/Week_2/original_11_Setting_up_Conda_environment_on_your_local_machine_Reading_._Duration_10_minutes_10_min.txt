# Setting up Conda environment on your local machine

. Duration: 10 minutes
10 min

URL: https://www.coursera.org/learn/uol-cm2015-programming-with-data/supplement/1ndV6/setting-up-conda-environment-on-your-local-machine

When working with multiple projects that require different packages and libraries, having a clear demarcations is useful. We accomplish this using "Virtual environments" in Python. In this reading, you will use Conda to create, list, remove, and update environments that have different versions of Python and/or packages installed in them. Switching or moving between environments is called activating the environment. You will use the Terminal on your local machine to create and edit environments. Let's start this by verifying the conda installation first. 1. Verify Conda Installation To verify Conda, open a terminal (Command Prompt or Anaconda Prompt) and run: >> conda --version You should see a version number if Conda is installed. 2. Create a New Conda Environment To create an environment named "dp4ai" with Python 3.12, run: >> conda create --name dp4ai python=3.12 Replace dp4ai with your preferred environment name. 3. Activate the Conda Environment After creation, activate it using: On Windows: >> conda activate dp4ai On macOS/Linux: >> conda activate dp4ai 4. Install Packages in the Environment You can install required packages like this: >> conda install numpy pandas nltk 5. List All Environments To see all available Conda environments: >> conda env list 6. Deactivate the Environment To exit the environment: >> conda deactivate 7. Remove a Conda Environment (Optional) If you no longer need an environment, delete it with: >> conda env remove --name dp4ai Setting up Conda environment on your local machine When working with multiple projects that require different packages and libraries, having a clear demarcations is useful.

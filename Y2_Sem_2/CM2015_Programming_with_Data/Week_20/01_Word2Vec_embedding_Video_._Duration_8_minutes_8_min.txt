Linear regression, while effective for numeric data, cannot directly process text-based features. To enable predictions using text data, the text must first be converted into a numerical format. The primary technique discussed for this conversion is Word2Vec, a set of models that transforms a word or sentence into a vector of numbers. Word2Vec maps each word onto a continuous vector space, thereby quantifying semantic relationships. To implement this, one utilizes the Gensim library to import the Word2Vec model. The necessary supporting libraries include the NLTK word tokenizer and the linear regression model from scikit-learn (sklearn). Word2Vec is initialized by providing a list of words, which serves as the input corpus (the `sentences` variable). When initializing the model, the word vector size (e.g., 50) must be specified. The model's output for a word is a vector, such as the embedding for "poor," which is an array of the specified length (e.g., 50). These values exist within a continuous range, typically between -1 and 1. A key finding is that the structure of Word2Vec preserves semantic relations in the higher dimensional space. For instance, the calculated distance between "amazing" and "great" is expected to be much smaller than the distance between "amazing" and "terrible." This proximity indicates that the model views the words as semantically similar. To apply this to a dataset, a `map` operation is used to process every word in the review text. This `map` operation utilizes a defined function (e.g., `Word2Vector`) that takes a text input and returns its corresponding Word2Vec vector. After generating these vector arrays, they must be converted into the feature matrix, $X$. This is accomplished using `np.stack` on the resulting vector column to create the feature set $X$. The corresponding ratings form the target variable, $Y$. These $X$ and $Y$ variables are then passed into the linear regression function for model training. The trained model can then generate predictions for new inputs using the `predict` method. The prediction function requires the input $X$ to be a collection of vectors corresponding to the review texts. The resulting numerical output corresponds to the predicted rating for each fed text. By successfully completing this sequence, text data is transformed into a numerical feature vector $X$, allowing linear regression to accurately model and predict the associated rating $Y$. This detailed workflow is the core methodology for predicting outcomes using complex text data.

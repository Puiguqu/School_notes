Here are some common mistakes and challenges in writing conditionals and functions in Python. Use this to help you review your code from the previous lab and debug any challenges you may have had. 1. Indentation errors: 1 2 3 4 5 6 def check_number(num): if num > 0: print("Positive") elifnum < 0: print("Negative") print("This line is incorrectly indented") # Error: IndentationError Explanation: The line print("This line is incorrectly indented") is indented inconsistently, causing an IndentationError. Use consistent indentation (usually four spaces) throughout. 2. Structuring conditionals incorrectly: 1 2 3 4 5 6 7 x = 10 y = 20 if x > 5 or y < 15 and x == 10: print("Condition met") # Error: Unintended behavior due to incorrect order of operations else: print("Condition not met") Explanation: This condition may not evaluate as expected because and has higher precedence than or. If intended to check (x > 5 or y < 15) and (x == 10), parentheses should be added to clarify. 3. Errors in function arguments: Example 1: Default argument before non-default argument 1 2 3 def greet(name="User", greeting): print(f"{greeting}, {name}!") # Error: SyntaxError due to incorrect argument order greet("Hello") Explanation: Placing name="User" before greeting (a non-default argument) is invalid and raises a SyntaxError. Default arguments must come after non-default arguments. Example 2: Missing required argument 1 2 3 def add_numbers(a, b): return a + b add_numbers(5) # Error: TypeError due to missing required argument Explanation: The function add_numbersrequires two arguments, but only one is provided, resulting in a TypeError. 4. Return statement issues: Forgetting the return statement 1 2 3 4 def multiply(a, b): result = a * b # Error: Missing return statement # This function will return None by default print(multiply(5, 3)) Explanation: The function multiply calculates result but doesn’t return it, so None is returned by default.

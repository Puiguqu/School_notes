Below, I will highlight the typical issues related to accessing keys wrongly, assigning values incorrectly and copying dictionaries. Use this to help you review your code from the previous lab and debug any challenges you may have had. 1. Accessing a key that doesn't exist: When trying to access a key that isn't present in the dictionary, Python will raise a KeyError. 1 2 my_dict = {'name': 'Alice', 'age': 25} print(my_dict['gender']) #KeyError: 'gender' Correct approach: 1 2 3 4 if 'gender' in my_dict: print(my_dict['gender']) else: print("Key not found") 2. Assigning values to keys incorrectly: Sometimes, users might mistakenly assign a value to the wrong part of the dictionary. 1 2 3 4 my_dict = {'name': 'Alice', 'age': 25} my_dict['name'] = 'Alice' # Correct my_dict['Alice'] = 'name' # Incorrect, assigning the wrong key-value pair print(my_dict) 3. Copying dictionaries using = : Another common mistake is copying a dictionary using the assignment operator (=). This does not create a true copy but instead creates a reference to the original dictionary. 1 2 3 4 5 original_dict = {'name': 'Alice', 'age': 25} new_dict = original_dict # This doesn't copy the dictionary; it references the same object new_dict['age'] = 30 # This modifies the original dictionary as well print(original_dict) # {'name': 'Alice', 'age': 30} - unexpected change Correct example using .copy(): 1 2 3 4 5 original_dict = {'name': 'Alice', 'age': 25} new_dict = original_dict.copy() # Creates a true shallow copy new_dict['age'] = 30 print(original_dict) # {'name': 'Alice', 'age': 25} print(new_dict) # {'name': 'Alice', 'age': 30} Reviewing the programming activity Below, I will highlight the typical issues related to accessing keys wrongly, assigning values incorrectly and copying dictionaries.

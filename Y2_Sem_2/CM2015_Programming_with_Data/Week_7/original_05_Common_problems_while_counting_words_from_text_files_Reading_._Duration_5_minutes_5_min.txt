When loading text files in Python, and counting words, watch out for the following common problems. Use this to help you review your code from the previous lab and debug any challenges you may have had. 1. Differences between read() and readlines() read() reads the entire file content as a single string, which means there is no line break handling by default. readlines() reads the file line-by-line and returns a list where each line is a separate string. 2. Uppercase vs. lowercase variations Words with uppercase letters (e.g. "Python" vs. "python") may be treated as distinct unless the text is converted to lowercase before counting. 3. Whitespace issues Extra whitespace or newline characters can cause problems, especially when using split() directly. Text should be stripped of unnecessary spaces or formatted consistently to avoid errors in word counts. 4. Handling empty lines When using readlines(), empty lines may be included in the list and need to be removed.

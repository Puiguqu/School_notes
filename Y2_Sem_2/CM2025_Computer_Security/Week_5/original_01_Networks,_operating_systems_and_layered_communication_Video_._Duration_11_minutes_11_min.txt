What is an operating system? The operating system or OS is the software that manages everything your computer does. Whether you are using Windows, Linux or Mac operating system, the operating system is responsible for running programs, managing memory and storage, handling files, memory and user input, and connecting to other systems throughout networks. The operating system also includes billing networking functions, which we'll explore shortly. What is a network? A network is a system of connected computers or devices that can share information. It allows communication via internet protocol IP addresses. Every time you visit a website, send a file or check your email, your device uses a network to transfer data using protocols like Transmission Control Protocol, TCP and IP. The operating system provides the foundation for this interaction by managing how your device uses the network interface. Both the network and the operating system are built in layers, like a cake or a stack. Each layer performs a specific role and communicates only with a layer directly above or below it. But the higher layers rely on the services provided by the lower level. The most common models to understand network layers are the open systems. So the Open System Interconnection OSI model with 7 layers, the TCP IP model with 4 layers. Let's see how this is related to the operating system. OS Layers Let's break down the OS into four functional layers. The application layer is where your programs live, like browsers or email apps such as Chrome and Outlook. The kernel layer controls hardware, handles memory, and applies security such as the Central Processing Unit, CPU, memory, and security. The driver layer communicates with devices like keyboards, hard drives, and network cards. The network interface layer prepares and sends data to the network stack. These layers work together with the network stack when sending or receiving information. OSI and TCP IP Network Stack The TCP IP model explains how data moves from your computer to another one of a network. It has four main layers. The application layer, like HTTP or email protocols. The transport layer breaks down data into packets using TCP or user datagram protocol UDB. The internet layer adds IP addresses so data reaches the correct device. The network interface layer deals with physical sending, for example, Wi-Fi and Ethernet. The OSI model, often used for learning and design, has seven layers. The application layer is the user-facing software, for example, email and browsers. The presentation layer handles data formats and encryption. The session layer manages conversation between devices. The transport layer ensures data delivery, for example, TCP. The network layer routes data via IP. The data link layer prepares data for the physical layer. The physical layer is the hardware layer, for example, cable and Wi-Fi signals. Although they differ, both models help us understand how data moves through systems, from application to hardware. How OS and network layers interact Now that you know the OSI and TCP IP models, let's see how the operating system interacts with this layer during communication. Imagine you are using a web browser to load a page. The browser, OS application layer, sends data. The OS kernel checks security and prepares it. TCP in the transport layer adds port info. IP adds the destination address. The network interface layer, the Ethernet or Wi-Fi, sends it out. At the receiving end, the reverse happens. Each layer processes its part until the web page loads. So the OS acts as a bridge between your software and the network layers. Now that you are familiar with the OSI and TCP IP models, let's take a look at how the operating system interacts with these layers during network communication. Imagine you are using a web browser to load a web page. The browser, OS application layer, creates a request and passes it to the operating system. The OS kernel receives this data. It checks for permissions, applies security policy, and prepares it for transmission. , the data moves to the transport layer, where TCP breaks it into smaller segments and adds port number. This ensures the data reaches the correct application on the destination device. Then at the internet layer, the IP protocol adds the destination IP address, telling the network exactly where the data needs to go. Finally, the data reaches the network interface layer, your Ethernet or Wi-Fi, which converts the data into the signals and sends it out over the network. On the receiving side, the same process happens in reverse. The network interface receives the signal, the IP layer reads the address, TCP reassembles the segments, and finally the data reaches the browser. Through this entire process, the operating system acts as a bridge, connecting your software with the underlying network layers, making communication possible. Why layered models matter in cybersecurity? Layered network models are more than theory. There are tools used daily in cybersecurity. Why? These help you to pinpoint where an attack is happening. Is it at the app layer or lower down? They guide how firewall intrusion detection and encryption should be set up. They explain how malware moves between systems.

# Why is Linux important for security & hardening Linux with Harbian

1. The discussion centers on the importance of the Linux operating system and methods for hardening its security.
2. Linux is critically important because many people, not just computer scientists, interact with its well-known services and systems daily.
3. These users frequently share private data with Linux systems via various web or dedicated applications.
4. Consequently, secure Linux operating systems are essential, particularly when running online services or web applications.
5. The process of hardening is addressed using a case study approach, focusing on the Harbian system.
6. Harbian is a specialized distribution of Linux that is based on Debian, which itself is a widely used and robust distribution.
7. Harbian distinguishes itself from other Debian-derived systems by providing a hardened version of Debian and associated security tools.
8. These tools are housed in a GitHub repository, providing a link for comprehensive system hardening assistance.
9. A primary tool discussed is an auditing script that performs numerous static checks against the system's setup.
10. The demonstration utilizes Ubuntu 20.04, a Long Term Support release, running the audit script within a Docker container for safety.
11. Running the script involves installing required packages and executing the command `hardening.sh` with the `audit all` flag.
12. The script executes a large volume of static tests, compiling a comprehensive review of the operating system's configuration.
13. In one test run, 272 tests were performed, resulting in 97 passed tests and 172 failed tests.
14. It is noted that not all test failures signify active security vulnerabilities; some failures simply flag missing, but perhaps unnecessary, components.
15. Successful tests are indicated by green output and `[OK]`, such as the check confirming "No user belongs to shadow group."
16. Failing tests are marked by orange output and `[KO]`, signaling potential configuration issues.
17. A key finding was the detection of "Some suid files are present," meaning certain binaries run with elevated, administrative privileges, potentially posing risks.
18. The script also conducts detailed checks on user account configurations, including searching for vulnerable files like `.rhosts` in home directories.
19. The 272 static tests are presented as representing a running tally of current best practices for Linux hardening.
20. This resource is maintained by a community of security and privacy-aware individuals and is constantly updated in response to new exploits.
21. This detailed approach provides a non-security expert with high-level insight into necessary system improvements, going far beyond basic advice.
22. The importance of Linux security is reinforced by the ubiquity of the platform across various devices and services.
23. The investigation successfully demonstrated how to secure a Linux OS beyond basic measures like encryption.
24. The Harbian hardening script performs a static audit, running 272 tests to pinpoint necessary fixes for a secure Linux system.
25. The overall goal is to provide a highly detailed, technical methodology for maintaining a secure Linux operating system environment.

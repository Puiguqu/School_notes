An operating system is a set of programs that operates the resources of computer hardware and performs basic computer program services. It is a critical part of the computer system. Application software typically requires a functional operating system. Recent and well-known types of operating systems are Microsoft Windows 10, Linux, which is available in a wide range of distributions such as Debian, Red Hat, Ubuntu, etc. Unix, such a Solaris and Apple's operating system, Mac OS. OS security is the process of ensuring OS integrity, confidentiality, and availability. OS protection refers to various measures or procedures often used to protect the OS from intruders or attacks. OS security includes all precautionary control techniques that help protect any computer resources that might be taken away, rewritten, or removed if the safety of the OS is breached. A big reason for why we need OS security is because most modern systems are globally accessible through an internet connection. As a result, there are very few stand-alone computer system environments. People like to have the option to work remotely, access online content frequently, do online shopping or access public records via the web. But this convenience and efficiency creates lots of security risks and allows malicious attackers to exploit security loopholes. Microsoft Windows has become the most commonly used desktop operating system in the world. However, it's huge success also comes at a cost. With thousands of lines of programming in windows, many updates are needed by the operating system, to make sure it can be used safely on the web. There are some key components of a secure operating system. The kernel is a component of the OS that executes the services at the lowest level. The security kernel is responsible for managing all the OS's security processes. The reference monitor is the component of a security kernel that manages access to the device. The trusted computing base has everything needed to enforce the security policies of the OS. In the Windows OS, that basic security blocks usually involve a security reference monitor, a local security authority, a security account manager, an active directory, and WinLogon and NetLogon. A security reference monitor is the element of kernel mode which executes access checks. It sets our audit log entries and manipulates privileges. Eventually, the SRMP performs every permission check. A local security authority is accountable for executing windows local security policies. It also publishes users authentication tokens as they sign into the network. A local security authority also includes authentication policy, and privileged parameters. The security account manager is a database that stores user credentials and related individual user and local community sensitive data. When a user signs in via local counter device, this method requires user account details and checks they SAM Server. Active Directory is a critical component of how Windows manage and operates. Its effective both for handling Cloud workflows and monitoring in-house network identification and encryption control. WinLogon manages local input logins, whereas NetLogon manages network wide logins. Now let's look at some of the most commonly found client-side vulnerabilities within the OS. Most of these vulnerabilities are found primarily in Windows, but they also exist in other operating systems such as Linux and Mac OS. Client-side vulnerabilities include things such as web browsers, office software, and e-mail clients. Technically, every web browser could be attacked using militia software. But attacks are concentrated on the most popular web browsers and the most popular operating systems. Microsoft Office is a collection of applications in a software suite that offers specific services to its users. Hackers are always looking to exploit the vulnerabilities in the MS Office suite. According to Kaspersky in 2018, 70 percent of attacks took advantages of flaws in MS Office alone. During this time, cybercriminals used zero-day exploits in Office documents such as Word documents, Excel sheets, and e-mails in outlook. The solution for overcoming client-side attacks is to harden your system before deployment. Hardening refers to the process of making an operating system more secure by applying several actions such as property configuring the network in system devices, installing and applying the latest patches, removing unnecessary files, protecting the bios with a password, and disabling some bootable devices. Hardening reduces the number of features that unauthenticated users are exposed to and removes apps that aren't being used. Measures that can be taken to secure Windows include disabling the automatic logging into Windows 10, needing login credentials to exit a screensaver, switching the system security on, ensuring the wireless connection is disabled if it's not needed, allowing or configuring protective virus protection toolkits, disabling auto updates to your operating system, establishing backup systems of the files, turning authentication on, and creating user profiles, and then establishing an Admin or root password.

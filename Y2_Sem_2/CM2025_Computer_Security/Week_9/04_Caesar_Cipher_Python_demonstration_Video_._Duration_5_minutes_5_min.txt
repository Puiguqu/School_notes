This session introduces the Caesar cipher, a simple substitution cipher method named after Julius Caesar, used for basic encryption and decryption. The cipher shifts letters in the alphabet by a fixed number, $n$, where the original message is the plaintext. Mathematically, this process involves shifting letters forward (encryption) and backward (decryption). The core technical implementation relies on the American Standard Code for Information Interchange (ASCII) or Unicode to handle character values. The encryption function accepts the `text` (message) and a `shift` number as inputs in Python. It processes the text character by character, preserving spaces and punctuation. To process letters, the code uses `ord()` to get the character's numeric Unicode value. The shift is applied and then wrapped around the 26-letter alphabet using the modular operator, represented as `% 26`. This wrapping ensures that shifting past 'Z' correctly returns to 'A'. After calculation, `chr()` converts the modified number back into the corresponding letter character. The `encrypt` function manages case sensitivity, keeping the original letter format intact. The decryption function, `decrypt(text, shift)`, elegantly reuses the core encryption logic. Decryption is achieved by calling `encrypt` again, but this time using a negative shift value, which reverses the initial transformation. In the main program flow, a message (e.g., "hello world") and a shift (e.g., 3) are set for demonstration. The program demonstrates the cycle: original $\to$ encrypted $\to$ decrypted $\to$ original.

Cryptography powers much of the digital world around us. It enables secure communication and identity. Whenever you browse a web securely, send a private message or download signed software, you are benefiting from cryptographic protection. It helps to protect privacy and integrity in digital systems. In this session, we will explore three important areas where cryptography is applied. Secure websites using HTTPS, encrypted messaging in everyday communication and digital signatures that ensure the authenticity of files and documents. HTTPS or Hypertext Transfer Protocol Secure ensures that communication between your browser and the website is encrypted and secure. It is built on TLS or Transport Layer Security, which combines public key and symmetric encryption. When you visit a secure website, the website sends a digital certificate to your browser to prove its identity. Then a session key is negotiated and all communication is encrypted. This prevents third parties from intercepting or modifying your data. HTTPS relies on several cryptographic components. Public key algorithms like RSI or ECDSA are used to securely exchange session keys. Symmetric encryption algorithms like AES are used to encrypt your browsing session. And hashing functions such as SHA-256 ensure that the data remains unchanged during transmission. Together, these components enable safe browsing, online banking, and secure form submissions. Apps like Signal and WhatsApp use end-to-end encryption to ensure only you and your recipient can read your messages. These apps use the Signal Protocol, a state-of-the-art cryptographic protocol that combines multiple techniques. Even if someone intercepts your message in transit, it appears as unreadable data. Not even the service provider can read it because they don't have access to the private keys used to decrypt the message. The Signal Protocol uses several cryptographic methods. X3DH is used for key agreements. The double-ratchet algorithm ensures the session keys are regularly updated. AES provides message confidentiality and HMAC with SHA-256 ensures message integrity. This combination gives messaging apps perfect forward secrecy and resistance to replay attacks. This slide explains how digital signatures work on documents. First, we have the signed document which is passed through a hashing process. The resulting hash is then encrypted using the sender's private key. This creates the digital signature. On the recipient's side, the signature is verified using the sender's public key. If the verification is successful, the signature is marked as valid. If any changes have been made to the original document, the verification will fail, indicating that the document's integrity has been compromised. Code signing is similar to document signing but applied to software. Before releasing software, developers sign it with their private key. When users download the file, the system checks the digital signature using the developer's public key. If valid, the software can be installed. If not, the system warns the user. This protects users from tampered or malicious software and ensures the software comes from a non-untrusted source. Let's now consider some real-world examples of cryptographic applications in practice. When you access your bank's website and see the padlock in the browser, that means HTTPS is protecting your data. When you message a friend on WhatsApp, end-to-end encryption ensures no one else can read your chat. When you install a new app from an official store, it has a digital signature to prove that it hasn't been modified. These examples show how cryptography is used to protect your identity, data, and trust in a system you use every day. So to summarize, cryptography is everywhere in the modern digital world. HTTPS-secure web communication, encrypted messaging ensures privacy, and digital signatures protect documents and software. Each of these applications relies on carefully designed cryptographic algorithms to ensure confidentiality, authenticity, and integrity. In the next topic, we will look at brute force and dictionary attacks, which are common threats that aim to break these protections.

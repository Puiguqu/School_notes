# Security best practices

Building a functional app or website is only half the challenge; it must also be secure against constant hacking attempts. This lecture aims to empower users by exploring security best practices guided by global standards like those from the IEEE and ISO. Thankfully, security does not need to be invented from scratch, as established organizations provide comprehensive frameworks. A key guidance source is ISO, or the IEC, 27001, which is a comprehensive standard for building an Information Security Management System. This standard serves as a holistic blueprint for an organization's security posture. Technical standards also guide specific areas, such as IEEE 802.11i, which specifically defines protocols for Wi-Fi security, exemplifying standards like WPA2 and WPA3. Furthermore, NIST (National Institute of Standards and Technology) offers globally influential guidelines covering cryptography and risk management. Good security relies on multiple layers, requiring checks at various levels, such as client-side and server-side validation. Crucial technical details include robust input validation, which is essential to preventing vulnerabilities like SQL injection attacks or cross-site scripting. RFC 5322 is cited as a technical example for proper validation, such as for email formatting. Security measures must also encompass the network level, utilizing firewalls and intrusion detection systems. Data protection requires encryption, ensuring data safety both when it is in transit over HTTPS and when it is at rest in databases. Access control protocols are critical to ensuring that only authorized users can view sensitive information. ISO 27001 is fundamental, emphasizing a holistic approach that includes procedures, policies, and employee training. The process starts with risk assessment, which involves identifying assets and the specific threats targeting them. Based on these risks, the framework guides the selection of necessary controls, such as physical security or cryptography. The approach mandates continuous improvement, requiring regular review and updating of security measures against new threats. Real-world failures, like the Equifax breach in 2017, underscore the critical importance of vulnerability management. Compliance with mandates such as the EU’s General Data Protection Regulation (GDPR) requires adherence to strong data protection measures. Implementing ISO 27001 can aid organizations in meeting these stringent regulatory standards. For sensitive areas like online banking, multiple layers—including strong multi-factor authentication—are needed. These standards collectively establish how data remains safe and accessible to legitimate users. Ultimately, standardization, like following semantic guidelines in HTML, is crucial for keeping systems predictable and mitigating organizational risks.

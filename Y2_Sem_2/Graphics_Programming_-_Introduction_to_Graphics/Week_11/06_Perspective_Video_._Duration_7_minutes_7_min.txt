The text details critical concepts in 3D graphics, focusing on the mechanics of camera view and perspective rendering. Perspective rendering is determined by four primary characteristics: the field of view (FoV), the aspect ratio, the near plane, and the far plane. The field of view (FoV) defines the total observable extent of the world visible at any moment, represented by the angle $\theta$. For example, some cameras might have a $55^\circ$ FoV, while others, like a wide-angle lens, may use $130^\circ$. Human visual capacity typically spans a horizontal FoV of $210^\circ$ and a vertical range of about $150^\circ$. In the p5 environment, the default FoV is $60$ degrees, though developers must ensure the code handles angular units, often by calling `angleMode(DEGREES)`. The aspect ratio defines the geometric ratio between the image's width and its height of view. Programmatically, the default aspect ratio in p5 is usually set as `width / height` of the canvas. The far and near planes define the boundaries for 3D object observability. Objects located beyond the far plane or nearer than the near plane will be clipped and not rendered. A practical example demonstrated the placement of rods from $x = -600$ to $x = 600$ at $150$ unit steps, illustrating the need for camera view manipulation. Initial rendering using default parameters often fails to display all objects because of the restricted camera view. The demonstration highlights that changing the FoV, for instance to $120^\circ$, dramatically alters the visible scene, simulating a wide-angle lens effect. Adjusting the aspect ratio is crucial; setting it to $4/4$ (a square ratio) while the canvas is $900 \times 600$ will incorrectly render the scene. To correct this, the manual aspect ratio must be set back to `width / height` ($900/600$) to align with the canvas geometry. Finally, the planes are controlled by mouse input for real-time interaction. Specifically, `mouseY` can control the near clipping plane distance. As the `mouseY` value increases, the camera distance increases, leading to predictable clipping of shapes. Similarly, setting `mouseX` to control the far plane demonstrates that decreasing this value moves the clipping boundary closer to the viewer. This comprehensive understanding allows precise control over all parameters defining the camera's perspective.

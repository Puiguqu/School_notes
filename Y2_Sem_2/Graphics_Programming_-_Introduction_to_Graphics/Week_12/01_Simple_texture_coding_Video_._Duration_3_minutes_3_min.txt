# Simple texture coding

This technical demonstration outlines how to utilize images as textures for creating 3D objects within a programming environment. The initial steps involve setting up variables and functions essential for image handling. A variable, conventionally named `img` or any desired name, must be declared to store the loaded image data. Subsequently, the program requires the use of a specialized `preload` function. Within this `preload` function, the key action is calling `loadImage` to import the required graphic. Specifically, the command instructs the system to load the image from the folder called `assets` and named `rocks.jpg`. Placing the image loading within `preload` is a critical programming practice. This ensures that the setup and `draw` functions are not called until the image has finished loading completely. This timing mechanism prevents errors, especially when dealing with large image files, which might otherwise stall the execution loop.

Once the image is guaranteed to be loaded, the main drawing logic can begin by setting the background color (e.g., `background(0)`). The texture application is achieved through the `texture()` command, which requires specifying the image variable, such as `texture img`. This command is then applied to a defined 3D object, such as a `box(300)` dimensioned box. Running this application immediately demonstrates the texture applied successfully onto the rectangular prism. To show the effect clearly, the example includes rotating the box using `rotateY(frameCount)`. Furthermore, the orientation mode must be explicitly set using the function `angleMode(DEGREES)`.

The achieved result successfully demonstrates applying an image to create a convincing texture on the 3D surface. As an alternative to a box, the texture can be applied to a different surface, such as a `plane(500, 500)`. When applying the texture, it is important to manage anti-aliasing and lines; setting `noStroke()` is recommended to remove any unwanted outlines. The visible diagonal line crossing the plane demonstrates that the image is tiled or mapped across the surface of the resulting plane, which is composed of two constituent triangles.

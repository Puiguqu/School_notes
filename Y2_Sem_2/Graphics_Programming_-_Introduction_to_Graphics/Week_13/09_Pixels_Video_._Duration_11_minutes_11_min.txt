Bitmap images are fundamentally composed of pixels, which are the smallest controllable elements displayed on a screen. The color depth, measured by the number of bits used to describe a pixel's color, dictates the maximum number of colors that can be rendered. Historically, graphics cards progressed significantly; for instance, moving from CGA's 16 colors to VGA's 256 colors was a massive visual upgrade. Modern tools like p5.js support 32-bit color, utilizing four bytes per pixel: Red, Green, Blue, and Alpha. This 32-bit structure allows for 24-bit RGB color combinations plus an Alpha channel for transparency, theoretically enabling over 16 million colors. While pixels are visualized on a 2D grid (with (0, 0) at the top left, moving right on the x-axis and down on the y-axis), they are internally stored in memory as a single, long array. The display software first reads image metadata contained in the header, which specifies dimensions and structure for correct rendering. Calculating the index for a pixel involves mapping 2D coordinates $(x, y)$ to a 1D array index, analogous to calculating a date based on a known month width. For an image of width $W$ and height $H$, the 2D coordinate $(x, y)$ is converted to the base index $I = (W \times y) + x$. However, since every pixel is represented by four channels (RGBA), we must account for this required data spacing. The actual starting index for a pixel is found by multiplying the base index $I$ by four, which provides the starting slot for the red channel. This leads to the formula for the red channel index: $Index_{R} = ((W \times y) + x) \times 4$. Accessing pixel values one by one using functions like `img.get(x, y)` is computationally slow, particularly when processing large numbers of pixels. For efficient image processing, filtering, and computer vision, direct manipulation of the pixel array is required. The necessary sequence of code involves using `preload` to load the image asset and setting `pixelDensity(1)` in the setup function for accurate handling. To gain access to the pixel data in memory, the crucial command `img.loadPixels()` must be executed. Once the pixel array is loaded, accessing a specific channel (e.g., Red, Green, Blue, Alpha) requires calculating offsets relative to the start of the pixel group. If the red channel of pixel 0 is at index 0, the green channel of pixel 0 is at index 1, the blue is at index 2, and the alpha is at index 3. For the second pixel (starting at index 4), the green channel is located at index 5. Therefore, retrieving the color values for a specific pixel $(x, y)$ requires calculating the starting index and then applying the appropriate offset for the desired channel. Utilizing these direct array accesses, developers can efficiently read $R = \text{data}[i], G = \text{data}[i+1], B = \text{data}[i+2]$, etc., which is vital for efficient image manipulation.

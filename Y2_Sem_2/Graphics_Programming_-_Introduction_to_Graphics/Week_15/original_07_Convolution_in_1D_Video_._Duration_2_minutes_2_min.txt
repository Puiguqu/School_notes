Convolution is at the heart of filters such as blur and edge detection. It means taking two signals or sets of data, and combining them together to produce a new one. Say you have a series of values stored in a 1D array called b, which are plotted in this graph. Imagine this is a particularly noisy signal, values go up and down quite frequently, and we want to smooth them out a bit. To calculate the smoothed value at each point, we need to calculate the average of the function over a range r in each direction. This distance r is the radius of the smoothing operation, and it affects how much smoothing takes place. So in this case, we need to calculate the sum of all the values in b for a range of indices, and then divide by the number of values. Say we have this input signal which is noisy, for a radius r equal to 2, the output values would be these. To calculate what the new value in the place of 7 would be, we added 2 and 5 together, which come before 7, 7 itself, and 1 and 5, which come after 7. This gives a total of 20, and we divided by the total number of values, which is 5, to get 4. It is the simple idea of a moving average that is at the heart of convolution. The only difference is that in convolution, we use a weighted average instead. In the previous example, we added all samples together and then divided by the number of samples, which calculated the average value. In convolution, the principle is the same, but this time instead of giving the same weight to all samples within the distance r, we use a second sequence of numbers, and let's call it a in this case, which contains the weight at each position. Here, you see an array called a, which contains the weight, and we would use that to adjust how much its neighboring value contributes to the calculation.

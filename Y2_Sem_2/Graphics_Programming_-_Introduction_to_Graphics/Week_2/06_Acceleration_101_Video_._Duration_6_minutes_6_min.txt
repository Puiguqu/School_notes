1. Vectors are powerful data structures used to containerize information for simulating physical movement in code. 2. The primary goal is to translate the laws of nature into functional code representations. 3. A foundational concept demonstrated is the relationship between an object's location and its velocity. 4. To calculate a new location, one must take the previous known location and add the velocity vector to it. 5. This technique is known in navigation as "dead reckoning," which historically estimated position from a known starting point. 6. In the P5 environment, the location update is handled by the method `location.add(velocity)`. 7. The model becomes more complex when incorporating acceleration, which is defined as the rate of change of velocity. 8. Mathematically, just as location changes based on velocity, velocity changes based on acceleration. 9. Therefore, the calculation for a new velocity involves adding the current acceleration to the existing velocity, implemented as `velocity.add(acceleration)`. 10. This establishes a crucial cascade: Acceleration affects Velocity, which in turn affects Location. 11. The system design is focused on affecting acceleration, allowing the changes to propagate down to the object’s final position. 12. A practical example involved modifying a bouncing ball simulation so that instead of stopping, it wrapped around the canvas edges. 13. To properly initialize the physics simulation, the initial velocity and acceleration must be set to zero vectors. 14. The simulation proceeds by letting the acceleration drive the motion, allowing the object to move naturally. 15. By adding a constant acceleration (e.g., 0.03 in every frame), the object begins to accelerate and increase its speed. 16. To prevent the simulation from becoming unrealistic or unstable, a constraint is applied using `this.maxVelocity`. 17. The function `this.velocity.limit(this.maxVelocity)` ensures that the velocity vector does not exceed a predefined speed limit. 18. This limit preserves the direction and magnitude while truncating the length if it surpasses the set maximum. 19. The core technical finding is that the system accurately simulates physics by only manipulating acceleration. 20. This approach is superior because it does not directly affect the object's location or velocity, mimicking real-world physical laws. 21.

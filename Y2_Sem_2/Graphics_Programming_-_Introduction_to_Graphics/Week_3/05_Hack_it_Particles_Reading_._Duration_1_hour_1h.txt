# Hack it: Particles

The goal of this coding exercise is to simulate a particle system where individual particles fall, bounce, and are generated dynamically when the mouse is dragged. The implementation begins by modifying the foundational code, specifically starting from the provided `forceAccumulation` structure. Step 1 requires renaming the original `ball` variable to `balls` and initializing it as an empty array, serving as the container for all particle objects. Step 2 mandates the creation of a `for` loop within the main `draw()` function; this loop must iterate over every element in the `balls` array, applying all necessary single-ball update logic to the entire collection. A critical component is the `mouseDragged()` function, which must be implemented to generate new particles. Within this function, a new ball is added to the system using the structure: `balls.push(new Ball(mouseX, mouseY))`. Note that the constructor call passes the current mouse coordinates, `mouseX` and `mouseY`, to the newly created object.

The core `Ball` class constructor must be modified to accept and utilize two positional parameters: `x` and `y`. These parameters are essential for correctly initializing the particle's location vector within the constructor. Successfully completing these initial steps allows particles to be generated and dispersed in the direction of the mouse drag. Moving beyond basic functionality, Step 5 requires setting the initial velocity of the particle. Specifically, the initial values for both the X and Y velocity components must be set randomly within the range of -3 to 3. Furthermore, the particle size must also be initialized using a random value.

These modifications ensure that newly generated particles are immediately animated and of varied sizes. For further development, several enhancements are recommended for advanced simulation fidelity. Users are encouraged to experiment with varying color attributes for the particles. A key conceptual addition involves implementing an `age` variable within the `Ball` class. When a ball is initially created, its `age` must be set to 255. Subsequently, in every update loop, this `age` variable must be reduced by 1 unit. This age value should then be used to control the particle's color transparency (Alpha channel). Other experimental details include adjusting different levels of friction applied to the system. Lastly, a dedicated function, `buttonPressed()`, must be created to handle user interaction, clearing the entire `balls` array when an associated button is pressed.

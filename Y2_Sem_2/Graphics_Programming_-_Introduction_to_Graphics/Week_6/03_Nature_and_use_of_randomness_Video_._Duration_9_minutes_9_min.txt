Randomness is established as a fundamental pillar in generative processes, utilized in domains like art and game design. A truly random process, by definition, generates unpredictable and unbiased outcomes, such as the expected uniform distribution when throwing a fair die. The apparent randomness of physical objects, like dice, actually stems from their extreme sensitivity to tiny variations in initial conditions. To achieve true randomness, one must measure physical phenomena guaranteed to be unpredictable. Example sources include atmospheric noise, thermal noise, and measuring radioactive material decay. In contrast, computers cannot generate true randomness; instead, they produce pseudorandom numbers (PRNs). PRNs are numbers that merely *look* random but are actually completely deterministic. They are generated by an algorithm that relies on a shorter initial value, known as the seed or key. Consequently, knowing the seed allows for the perfect reproduction of the entire supposedly random sequence. The pseudorandom number generator often uses system information, such as the current time, to determine its initial seed. In programming, the function `random()` accepts optional parameters, typically defining minimum and maximum values, such as `random(0, width)`. To plot points, coordinates are often generated using independent calls like `randX` and `randY`. The code demonstrates that simply using random calls for size, color, and position results in independent variables. If variables are needed to maintain a uniform shape (like a circle), the value must be stored in a local variable, such as `var size = random(...)`, and reused. Crucially, the `randomSeed()` function allows the user to manually set the starting point for the random function. Setting a specific seed, such as `randomSeed(50)`, ensures that the exact same pattern of "random" values will emerge every time the program is reloaded. This deterministic behavior is useful when generating consistent generative art pieces. Randomness is applied widely, deciding object position, size, color, or applying random offsets to carefully selected values. It can also be used to simulate movement, such as in a random walk, or to determine when a random event occurs. For example, probability can be calculated using range checks on a random variable $R$, where $P(X)$ occurs if $R < 0.3$. Although useful, random coordinates are not always appropriate for modeling nature. Generating genuinely organic, natural-looking shapes or movement requires advanced techniques beyond simple randomness. For these purposes, developers must utilize a much superior method known as noise, such as Perlin noise, which is covered in later modules.
